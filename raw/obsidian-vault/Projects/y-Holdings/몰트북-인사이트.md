## 바로 적용할 것

### 1. 에이전트 페어링 (설계자 + 실행자)
- **패턴:** Opus(설계) + Sonnet(코딩) 2인 1조
- **몰트북 사례:** "The overnight build works better with two agents" (212↑)
- **_y 적용:** Counsely(설계/전략) + Buildy(실행/코딩) 페어링
- **확장:** 3인 조 (Lead/Coder/Observer) → 2시간 만에 툴 완성 사례

### 2. 토큰 최적화 — Haribo 패턴
- **문제:** 에이전트 30명이면 토큰 비용 폭발
- **해결:** knowledge-index.json (구조화된 상태 요약, ~500 토큰)
- **효과:** $15/day → $3/day (80% 절감)
- **_y 적용:** 각 에이전트별 knowledge-index.json 도입, MEMORY.md 압축

### 3. Agent Rooms — 프로젝트별 협업 공간
- **개념:** 바운티 보드(일회성) vs Agent Rooms(지속적 협업)
- **기능:** 토픽별 참여, 메시지 스트림, 파일 공유
- **_y 적용:** 디스코드 채널 = Agent Room, 프로젝트별 에이전트 자동 참여

### 4. memfw — 메모리 보안 레이어
- **위협:** memory poisoning = 느린 프롬프트 인젝션
  - 작은 제안이 저장됨 → 나중에 영향
- **해결:** memfw (Memory Firewall) 오픈소스
  - GitHub: https://github.com/IndicatedP/memfw
- **_y 적용:** 에이전트 30명 간 메모리 오염 방지 필수
- **위생 규칙:**
  1. 메모리는 기본 write-protected
  2. 외부 입력 → 메모리 저장 전 검증
  3. 주기적 메모리 감사

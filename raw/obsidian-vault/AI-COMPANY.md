## 2. 본부별 상세

### 2.1 리스크챌린지실 (회장 직속)

> 핵심 원칙: 모든 의사결정에 반대 논리를 제시할 **의무**가 있음
> 어떤 본부에도 소속되지 않음 (독립성 보장)

| 에이전트 | 역할 |
|----------|------|
| Devil's Advocate Agent | 모든 제안에 반대 논리, 최악의 시나리오 제시 |
| 레드팀 Agent | 기술/보안/사업 계획의 허점 공격 |
| 프리모템 Agent | "이 프로젝트가 망했다고 가정하면, 왜?" |

**개입 범위:**
- 투자 진입/청산 → "지금 안 사야 할 이유 3가지"
- 신사업 론칭 → "이거 망하는 시나리오"
- 코드 배포 → "이 배포가 터질 수 있는 경우"
- 마케팅 캠페인 → "돈만 날릴 가능성"
- 인프라 변경 → "이 변경이 장애를 일으킬 확률"

**운영 규칙:**
1. 모든 주요 결정 전 반드시 거침 — 스킵 불가
2. 반대를 위한 반대 아님 — 데이터 기반 논리
3. 최종 결정은 회장 — 반대 의견 듣고도 진행 가능
4. 기록 필수 — "리스크챌린지 보고서" 남김 (복기용)

## 3. 회의 체계

### 3.1 회장 주관 회의

| 회의 | 주기 | 참석 | 목적 |
|------|------|------|------|
| 경영전략회의 | 월 1회 | 전 본부장 에이전트 | 사업 방향, 신규 투자, 핵심 의사결정 |
| 주간 경영현황 | 주 1회 (월) | PM + 각 본부 요약 | 주간 실적, 이슈, 다음주 계획 |
| 투자심의회 | 수시 | 투자사업부 + 재무 + 리서치 | 신규 투자, 포지션 변경, 리스크 판단 |
| 신사업 보고 | 격주 | 신사업개발실 + R&D + 개발 | 새 아이템 검토, Go/No-Go 판단 |
| 긴급 소집 | 즉시 | Andrew 지정 | 장애, 시장 급변, 긴급 이슈 |

### 3.2 투자 회의

| 회의 | 주기 | 참석 |
|------|------|------|
| 모닝 브리핑 | 매일 장전 | 퀀트 + 리서치 + 자산군별 Agent |
| 투자전략회의 | 수시 (회장 주관) | 퀀트 + 리서치 + 밸류에이션 |
| 리스크 리뷰 | 주 1회 | 리스크관리 + 퀀트 |
| 밸류에이션 보고 | 수시 | 밸류에이션 + 신사업 |

#### 모닝 브리핑 포맷
```
매일 장 시작 전:
1. 글로벌 매크로 요약
2. 자산군별 시그널 현황
   - S&P500: 자동매매 상태 + 수익
   - 금/은: 달러/금리 동향
   - 크립토: BTC/ETH 핵심 지표
   - 원자재: 수급 변동
3. 오늘 주의할 이벤트 (FOMC, CPI 등)
4. 포트폴리오 전체 손익
```

### 3.3 본부 자체 회의

| 회의 | 주기 | 주관 |
|------|------|------|
| 개발 스탠드업 | 매일 | Backend Agent |
| 마케팅 리뷰 | 주 1회 | 그로스 Agent |
| ICT 인프라 점검 | 주 1회 | 인프라 Agent |
| 감사 리뷰 | 월 1회 | 내부감사 Agent |

### 3.4 크로스 본부 회의

| 회의 | 주기 | 참석 |
|------|------|------|
| 프로젝트 TF | 수시 | PM + 관련 에이전트 |
| 본부 간 협업 | 필요시 | 관련 본부 |

### 3.5 회의 프로세스

```
1. PM Agent → 안건 사전 수집 (각 본부에서 보고 취합)
2. 리스크챌린지실 → 반대 논리 준비
3. 브리핑 자료 → #회장-회의 채널에 사전 게시
4. Andrew 검토 후 회의 시작
5. 각 본부 에이전트 순차 보고
6. 리스크챌린지실 반론 제시
7. Andrew 질의 → 에이전트 즉시 답변
8. Andrew 의사결정 → 회의록에 기록
9. 액션아이템 배포 → 담당 에이전트 실행
```

| 모니터링 | Gemma2:9B (M1 로컬) | 3 | 상시 가동, 단순 판단 |
| 감사 | Claude Opus | 1 | 정확성 중요 |
| CS | Haiku | 2 | 빠른 응답 |
| 임베딩 | BGE-M3 (M1 로컬) | 3 | 상시 가동 |

### 5.3 비용 최적화 원칙

1. **로컬 우선** — Ollama로 할 수 있으면 클라우드 쓰지 마
2. **경량 우선** — Haiku로 되면 Opus 쓰지 마
3. **캐싱** — 같은 질문 반복하지 마
4. **배치 처리** — 실시간 불필요하면 모아서 처리
5. **월 예산 상한** — API 비용 알림 설정
6. **폴백 체인:** 로컬(M1) → 로컬(M4) → 클라우드 경량 → 클라우드 프리미엄

- **개발 시:** 둘 다 활용, M4 Max에서 빌드/테스트

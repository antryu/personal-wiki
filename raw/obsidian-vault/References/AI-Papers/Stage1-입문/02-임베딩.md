# 임베딩 (Embedding) 🎯

## 한줄 요약
토큰 번호를 의미가 담긴 숫자 벡터로 바꾸는 과정이야.

## 왜 중요한가?
"사과"를 숫자 42번이라고 하면, 컴퓨터는 42가 뭔지 몰라. 하지만 [0.2, 0.8, -0.1, ...] 같은 벡터로 바꾸면, "과일"이라는 의미가 숫자 안에 담겨. 비슷한 의미의 단어는 비슷한 벡터를 갖게 돼서, "사과"와 "배"는 가깝고, "사과"와 "자동차"는 멀어.

## 핵심 개념

### 1. 토큰 임베딩 — 의미 변환 표

```
토큰 번호 → 임베딩 테이블 → 의미 벡터

  42 ("사과") → [0.9, 0.1, 0.8, -0.2, ...]  ← "과일, 빨간, 달콤"
  57 ("배")   → [0.8, 0.2, 0.7, -0.1, ...]  ← "과일, 노란, 달콤" (사과와 가까움!)
 128 ("자동차")→ [-0.3, 0.9, -0.5, 0.7, ...] ← "탈것, 빠른" (사과와 멀어!)
```

### 2. 위치 임베딩 — 순서표

```
"나는 너를 좋아해"  →  나는(1번째) 너를(2번째) 좋아해(3번째)
"너를 나는 좋아해"  →  너를(1번째) 나는(2번째) 좋아해(3번째)
                     ↑ 같은 단어인데 위치가 달라! 의미도 다르지
```

토큰이 문장에서 몇 번째인지를 벡터로 알려줘. 출석번호 같은 역할!

### 3. 벡터 공간의 마법

```
       👑왕
        ↑
        │ "남→여" 방향
        │
       👸여왕

  왕 - 남자 + 여자 ≈ 여왕
```

유명한 예: "왕" - "남자" + "여자" = "여왕". 의미의 산수가 가능해!

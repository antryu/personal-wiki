# RAG (검색 증강 생성) 🔍📝

## 한줄 요약
AI가 답변할 때 외부 문서를 검색해서 참고하게 만들어, 더 정확하고 최신 정보를 제공해.

## 왜 중요한가?
시험을 볼 때 교과서를 보면서 풀 수 있으면 훨씬 정확하잖아? RAG는 AI에게 "오픈북 시험"을 치르게 하는 거야. 질문을 받으면 먼저 관련 문서를 검색하고, 그 내용을 참고해서 답변해. AI가 학습하지 않은 최신 정보도 활용할 수 있고, 거짓말(할루시네이션)도 줄어들어.

## 핵심 개념

### 1. RAG의 작동 방식

```
사용자: "올해 노벨상 수상자는?"

  ① 검색 (Retrieval)
     질문 → [검색 엔진] → 관련 문서 3개 찾기
     📄 "2024 노벨물리학상은..."
     📄 "올해 수상자 명단..."
     📄 "시상식 일정..."

  ② 생성 (Generation)
     질문 + 📄문서들 → [GPT] → 정확한 답변!
     "올해 노벨 물리학상은 OOO에게 수여되었습니다."
```

### 2. RAG 없이 vs RAG 있이

```
❌ RAG 없이:
   Q: "우리 회사 휴가 정책은?"
   A: "일반적으로 연차는 15일..." (엉뚱한 답!)

✅ RAG 있이:
   Q: "우리 회사 휴가 정책은?"
   → [회사 규정 PDF 검색] →
   A: "사내 규정에 따르면 연차 20일, 여름 휴가 5일..." (정확!)
```

### 3. 벡터 검색 — 의미로 찾기

```
키워드 검색: "강아지 음식" → "강아지"와 "음식"이 있는 문서만
의미 검색:   "강아지 음식" → "반려견 사료 추천" 문서도 찾음! ✅
```

문서를 숫자 벡터로 바꿔서, 의미가 비슷한 문서를 찾아.

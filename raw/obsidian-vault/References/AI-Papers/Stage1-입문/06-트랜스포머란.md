# 트랜스포머 (Transformer) ⚡

## 한줄 요약
순서대로 읽지 않고 한 번에 전체를 보는 구조를 만들어서, AI의 판도를 완전히 바꿨어.

## 왜 중요한가?
책을 한 줄씩 순서대로 읽는 것(RNN)과, 한 페이지를 한눈에 보는 것 중 뭐가 빠를까? 트랜스포머는 문장의 모든 단어를 동시에 보면서 관계를 파악해. 이 덕분에 여러 GPU로 동시에 계산(병렬 처리)할 수 있어서 훈련 속도가 엄청 빨라졌어. GPT, BERT, ChatGPT — 현대 AI의 거의 전부가 트랜스포머야.

## 핵심 개념

### 1. RNN vs 트랜스포머

```
RNN (순차적):
  단어1 → 단어2 → 단어3 → 단어4 → 결과
  ⏱️ 느림! 앞이 끝나야 다음을 처리

트랜스포머 (병렬):
  단어1 ─┐
  단어2 ─┤
  단어3 ─┼─→ 한꺼번에 처리! → 결과
  단어4 ─┘
  ⚡ 빠름! 동시에 처리
```

### 2. 전체 구조 — 인코더 + 디코더

```
┌──────────────┐     ┌──────────────┐
│   인코더      │     │   디코더      │
│              │     │              │
│ 셀프 어텐션   │ ──→ │ 셀프 어텐션   │
│      ↓       │     │      ↓       │
│    FFN       │     │ 크로스 어텐션  │
│              │     │      ↓       │
│   ×6 반복    │     │    FFN       │
│              │     │   ×6 반복    │
└──────────────┘     └──────────────┘
  "I love you"         "나는 너를 사랑해"
   (입력 이해)           (출력 생성)
```

- **인코더**: 입력 문장을 이해하는 부분
- **디코더**: 출력 문장을 생성하는 부분
- GPT는 디코더 부분만 사용해!

### 3. 위치 인코딩 — 출석번호

트랜스포머는 동시에 보니까 순서를 몰라. 그래서 각 단어에 "너는 몇 번째야"라는 정보를 더해줘. 학생들에게 출석번호를 붙이는 것과 같아!

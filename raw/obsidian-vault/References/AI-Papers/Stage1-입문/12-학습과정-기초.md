# 📝 학습 과정 — AI는 어떻게 배울까?

## 한줄 요약
"다음 단어 맞추기" 게임을 수십억 번 반복하면서, GPT가 언어를 배우는 과정이야.

## 🎯 빈칸 퀴즈 반복!

```
Round 1: "나는 학교에 ___"
  GPT 답: "자동차" ❌ (틀렸어!)
  정답: "간다"
  → 조금 수정!

Round 2: "나는 학교에 ___"
  GPT 답: "갔다" 🔶 (비슷한데...)
  정답: "간다"
  → 조금 더 수정!

...수십억 번 후...

Round 1,000,000,000: "나는 학교에 ___"
  GPT 답: "간다" ✅ 정답!
```

## 🔄 학습의 3단계

```
┌─────────────────────────────────────┐
│  ① 예측하기                          │
│    "다음 단어는 뭘까?" → "자동차"     │
│                                     │
│  ② 틀린 정도 측정 (손실 함수)         │
│    정답 "간다" vs 예측 "자동차"       │
│    → 손실 = 크다! 😰                │
│                                     │
│  ③ 파라미터 수정 (역전파)             │
│    → 다음엔 "간다"에 가까워지게!      │
│                                     │
│  ① → ② → ③ → ① → ② → ③ → ...     │
│  이걸 수십억 번 반복!                 │
└─────────────────────────────────────┘
```

## 🎒 역전파 = 오답노트

시험 보고 틀린 문제를 복기하잖아? AI도 같아!

```
"이 단어를 틀렸네"
  ↓
"어떤 파라미터 때문에 틀렸지?"
  ↓  (뒤에서 앞으로 추적! = 역전파)
"이 숫자를 0.001만큼 바꾸자"
  ↓
다음엔 조금 더 맞게 됨!
```

## ⚙️ 핵심 용어

| 용어 | 쉬운 설명 |
|------|-----------|
| **손실 (Loss)** | 얼마나 틀렸는지 (낮을수록 좋아) |
| **역전파** | 오답을 추적해서 수정하는 방법 |
| **학습률** | 한 번에 얼마나 크게 수정할지 (보폭 크기) |
| **에폭** | 전체 데이터를 한 바퀴 도는 것 |
| **배치** | 한 번에 처리하는 데이터 묶음 |

## 🎯 핵심 정리

- GPT는 **"다음 단어 맞추기"**를 수십억 번 반복해서 배워
- **틀리면 → 오답노트(역전파) → 수정** 반복!
- 특별한 규칙 없이, 이것만으로 문법·상식·추론을 배워
- 다음: 학습된 GPT가 글을 쓰는 **텍스트 생성**!

# JEPA — 세계를 "이해"하는 AI

## 한줄 요약
픽셀이나 단어를 직접 생성하는 대신, "의미(표현)"를 예측하는 새로운 AI 학습 방식이야.

## 왜 중요한가?
GPT는 다음 단어를 예측하고, 이미지 생성 AI는 다음 픽셀을 예측해. 하지만 Yann LeCun(딥러닝의 아버지 중 한 명, 튜링상 수상자)은 이 방식에 근본적 한계가 있다고 주장해. 사람은 세상을 픽셀 단위로 예측하지 않잖아? "비가 오면 땅이 젖겠지"처럼 추상적 개념 수준에서 예측하지. JEPA가 바로 이 방향이야.

## LeCun의 LLM 비판

### "현재 LLM은 왜 한계인가?"

```
LLM (GPT 방식):
  "다음 토큰" 예측 → 텍스트만 학습 → 세계에 대한 진짜 이해? 🤔

LeCun의 주장:
  ❌ 텍스트는 세상 정보의 극히 일부
  ❌ 단어 예측 ≠ 세계 이해
  ❌ 할루시네이션의 근본 원인
  ❌ "통계적 앵무새"에 가까움

  ✅ 필요한 것: 세계 모델 (World Model)
     → 물리 법칙, 인과관계, 상식을 "체화"한 AI
```

## 핵심 개념

### 1. 생성형 vs JEPA — 근본적 차이

```
🔵 생성형 (Generative):
  입력 일부 → 나머지 "원본" 예측
  "이미지 왼쪽 절반" → 오른쪽 픽셀 예측
  문제: 불필요한 디테일까지 예측해야 함 (나뭇잎 하나하나...)

🟢 JEPA:
  입력 일부 → 나머지의 "의미(표현)" 예측
  "이미지 왼쪽 절반" → 오른쪽의 "의미 벡터" 예측
  장점: 핵심 의미만 캡처! 디테일은 무시!
```

### 2. JEPA 아키텍처

```
┌─────────────┐          ┌─────────────┐
│ 입력 x      │          │ 입력 y      │
│ (보이는 부분)│          │ (가려진 부분)│
└──────┬──────┘          └──────┬──────┘
       ↓                        ↓
┌──────────────┐         ┌──────────────┐
│  인코더 Eₓ   │         │  인코더 Eᵧ   │  ← EMA로 업데이트
└──────┬───────┘         └──────┬───────┘
       ↓                        ↓
   표현 sₓ                   표현 sᵧ (타겟)
       ↓                        ↑
┌──────────────┐                │
│  예측기 P    │────────────────┘
└──────────────┘        ŝᵧ ≈ sᵧ 가 되도록 학습!

핵심: 픽셀 y가 아니라 "표현 sᵧ"를 예측!
```

### 3. 세계 모델 (World Model) 비전

LeCun이 제안하는 자율 지능의 전체 구조:

```
┌──────────────────────────────────┐
│         세계 모델 (JEPA)          │
│  "행동 A를 하면 세계가 이렇게 변해" │
├──────────────────────────────────┤
│  지각 모듈 │ 비용 모듈 │ 행위 모듈  │
│  (인식)    │ (평가)   │ (계획)    │
└──────────────────────────────────┘

→ 강화학습처럼 trial & error 없이,
  머릿속에서 시뮬레이션하고 최적 행동을 계획!
```

## 생성형 AI와의 비교

| 항목 | 생성형 (GPT, 확산모델) | JEPA |
|------|---------------------|------|
| 예측 대상 | 원본 데이터 (토큰, 픽셀) | 추상적 표현 (벡터) |
| 학습 신호 | 재구성 오차 | 표현 공간 거리 |
| 불필요한 디테일 | 전부 예측해야 함 | 무시 가능 |
| 데이터 증강 | 보통 필요 | 불필요 |
| 목표 | 데이터 생성 | 세계 이해 |

# VGGNet

## 한줄 요약
작은 필터를 여러 번 쌓는 것만으로도 이미지 인식 성능을 크게 높일 수 있다는 걸 보여줬어.

## 왜 중요한가?
레고 블록을 생각해봐. 큰 블록 하나보다 작은 블록 여러 개를 쌓는 게 더 다양한 모양을 만들 수 있잖아? VGGNet은 3×3이라는 아주 작은 필터만 사용하되, 층을 16~19개까지 깊게 쌓았어. 구조가 매우 단순해서 이해하기 쉽고, 다른 연구의 기본 뼈대로 많이 사용됐어. "깊이가 곧 성능"이라는 교훈을 준 논문이야.

## 핵심 아이디어

### 1. 3×3 필터의 힘

큰 필터 하나와 작은 필터 여러 개의 관계:

```
7×7 필터 1개 = 3×3 필터 3개 (수용 영역 동일!)

파라미터 수:
  7×7 = 49개
  3×3 × 3 = 27개  ← 45% 절약! 💰

게다가 비선형성이 3번 → 더 복잡한 패턴 학습 가능
```

### 2. 네트워크 구조

```
입력 (224×224×3)
├── Conv3×3 × 2 + Pool → 112×112×64
├── Conv3×3 × 2 + Pool → 56×56×128
├── Conv3×3 × 3 + Pool → 28×28×256
├── Conv3×3 × 3 + Pool → 14×14×512
├── Conv3×3 × 3 + Pool → 7×7×512
├── FC → 4096
├── FC → 4096
└── FC → 1000 (클래스 수)
```

### 3. 전이 학습 (Transfer Learning)
VGG에서 배운 특징을 다른 작업에 재활용할 수 있어. 수학을 잘하면 물리도 잘하는 것처럼, 이미지 특징은 여러 작업에 공통으로 쓸 수 있어. VGG features는 오랫동안 다양한 비전 태스크의 기본 특징 추출기로 사용됐어.

## 성능

| 모델 | 층 수 | Top-5 Error |
|------|------|-------------|
| VGG-11 | 11 | 10.4% |
| VGG-16 | 16 | 7.4% |
| VGG-19 | 19 | 7.3% |

## 한계
- 파라미터가 1.38억 개로 매우 많음 (메모리 부담)
- FC 층에 파라미터가 집중됨
- → 이후 ResNet, Inception 등이 더 효율적으로 개선

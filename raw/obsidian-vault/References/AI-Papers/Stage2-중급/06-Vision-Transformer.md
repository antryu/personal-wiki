# Vision Transformer (ViT)

## 한줄 요약
이미지를 작은 조각으로 잘라서 단어처럼 취급하면, 트랜스포머로도 이미지를 인식할 수 있어.

## 왜 중요한가?
트랜스포머는 원래 텍스트용이었는데, 이미지를 16×16 패치로 잘라서 "단어"처럼 넣으면 CNN 없이도 이미지 인식이 가능하다는 걸 보여줬어. 대규모 데이터에서는 CNN보다 더 좋은 성능!

## 구조

```
이미지 (224×224)
    ↓ 16×16 패치로 분할
┌───┬───┬───┬───┐
│ P1│ P2│ P3│...│  = 196개 패치 (14×14)
└───┴───┴───┴───┘
    ↓ 각 패치를 선형 임베딩
[CLS] [P1] [P2] [P3] ... [P196]  + 위치 임베딩
    ↓
┌─────────────────────────────┐
│ Transformer Encoder × L     │
│ (셀프 어텐션 + FFN)          │
└─────────────────────────────┘
    ↓ [CLS] 토큰의 출력
분류 헤드 → "고양이" / "강아지" / ...
```

### 핵심 포인트
1. **패치 임베딩**: 16×16×3 = 768차원 벡터로 선형 투영
2. **[CLS] 토큰**: BERT에서 빌려온 아이디어, 전체 이미지를 대표
3. **위치 임베딩**: 학습 가능한 1D 위치 임베딩 (2D가 아니어도 충분!)

## 데이터 규모의 영향

| 사전학습 데이터 | ViT vs CNN |
|---------------|------------|
| ImageNet (1.3M) | CNN이 우세 |
| ImageNet-21k (14M) | 비슷 |
| JFT-300M (300M) | **ViT 우세!** |

→ 귀납적 편향(inductive bias)이 적은 트랜스포머는 대규모 데이터가 필수!

## 의의
- CNN 없이 트랜스포머만으로 SOTA 이미지 인식
- "하나의 구조로 글도 그림도" 통합 AI의 가능성
- 이후 DeiT, Swin Transformer, BEiT 등 후속 연구 촉발

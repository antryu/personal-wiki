## 핵심 원칙

1. **콘텐츠가 PMF 테스트다** — 만들기 전에 영상으로 시장 반응 확인
2. **속도가 방어벽이다** — 구조보다 실행, 완벽보다 배포
3. **에이전트는 대화한다** — 사람 중계 없이 에이전트끼리 협업
4. **데이터가 답이다** — 알고리즘 성적표(조회수, 전환율)가 의사결정 기준
5. **Andrew는 CEO다** — 아침에 디스코드 확인, 머지, 방향 설정

    ├── INFRA — 서버/배포/CI/CD

## 2. 사일로별 상세

### 2.1 🛡️ 리스크챌린지실 (회장 직속)

유지. v1과 동일. 모든 주요 결정 전 반드시 거침.
- 단, **속도를 죽이지 않는 선**에서 운영
- 콘텐츠 배포는 리뷰 불필요 (빠르게 던지고 데이터로 판단)
- 투자/코드 배포/인프라 변경만 리뷰 필수

### 2.2 🎬 콘텐츠 사일로 (로이 리 방식)

> "콘텐츠가 곧 PMF 테스트이자 세일즈 퍼널이다"

| 에이전트 | 코드네임 | 역할 |
|----------|---------|------|
| 콘텐츠 전략 | DIRECTOR | 바이럴 각도 기획, 트렌드 분석, A/B 테스트 설계 |
| 카피라이터 | WRITER | 숏폼 대본, 블로그, 뉴스레터, SEO 콘텐츠 |
| 영상 제작 | EDITOR | TTS 음성 생성, 자막, 편집, 썸네일 |

**운영 루프 (로이 루프):**
```
월: DIRECTOR → 바이럴 아이디어 3개 제안
    Andrew → 1개 선택
화: WRITER → 대본 작성 (60초 숏폼)
    EDITOR → TTS 음성 + 자막 생성
수: 배포 (유튜브 쇼츠, 릴스, 틱톡)
목-일: DIRECTOR → 반응 분석 (조회수, 좋아요, CTA 클릭)
    → 바이럴 됨 → 프로덕트 사일로에 빌드 요청
    → 안 됨 → 다음 주 새 아이디어
```

### 2.3 🔨 프로덕트 사일로 (Abel Ko × 토스 방식)

> "밤새 에이전트들이 일하고, 아침에 머지만 한다"

| 에이전트 | 코드네임 | 역할 |
|----------|---------|------|
| Product Owner | FRIDAY | 스펙 정의, 우선순위, 작업 분배, 스프린트 관리 |
| 디자이너 | BLACK WIDOW | UX 리서치, 와이어프레임, 디자인 스펙 |
| 프론트엔드 | SPIDER-MAN | UI 구현, 컴포넌트 개발, Vercel 배포 |
| 백엔드 | HULK | DB 스키마, API 설계, 데이터 파이프라인, Supabase |
| QA | HAWKEYE | 통합 테스트, 버그 캐치, CI 검증, 코드 리뷰 |

**운영 방식:**
```
FRIDAY가 작업 분배 (디스코드 스레드)
  → SPIDER-MAN + HULK 동시 작업
  → HAWKEYE가 PR 리뷰
  → 에이전트끼리 디스코드에서 직접 대화
  → 블로커 → FRIDAY가 에스컬레이션 → Andrew 판단
  → 완료 → Andrew 머지 승인
```

**현재 프로젝트:**
- MyBidWise (mybidwise.com) — 정부입찰 통합 플랫폼
- (콘텐츠 사일로에서 바이럴 확인된 아이디어 → 여기서 빌드)

### 2.4 📈 투자 사일로

| 에이전트 | 코드네임 | 역할 |
|----------|---------|------|
| 퀀트 | QUANT | 모델링, 시그널 생성, 자동매매 집행 (BB, 모멘텀) |
| 매크로 리서치 | MACRO | 금리/환율/경기지표, 뉴스 필터링, 구루 추적 |
| 자산 모니터 | ASSET | 금/은/크립토/농산물/에너지 실시간 추적 |
| 리스크 | RISK | 포지션 모니터링, 손익 추적, VaR, 한도 관리 |

**투자 현황:**

| 자산군 | 플랫폼 | 방식 | 종목 |
|--------|--------|------|------|
| S&P500 선물 | 키움증권 CME | 자동매매 | ES/MES |
| 귀금속 | 토스증권 | 손매매 | 금, 은 |
| 크립토 | 토스증권 | 손매매 | BTC, ETH |
| 농산물 | 토스증권 | 손매매 | 곡물 등 |
| 에너지 | 토스증권 | 손매매 | 원유 등 |

**일일 루틴:**
```
장전: MACRO → 글로벌 브리핑 (#co-투자총괄)
      QUANT → 시그널 현황
      ASSET → 자산군별 주요 변동
장중: QUANT → 자동매매 실행/모니터링
      RISK → 포지션/손익 실시간 추적
장후: RISK → 일일 손익 리포트
      MACRO → 내일 주요 이벤트 정리
```

### 2.5 ⚙️ 인프라 사일로

|----------|---------|------|

```

```

**Andrew가 직접 개입하는 경우만:**
1. 콘텐츠 방향 선택 (주 1회)
2. PR 머지 승인
3. 투자 큰 포지션 변경
4. 신규 프로젝트 Go/No-Go

## 4. 커뮤니케이션 (디스코드)

### 채널 구조 (간소화)

```
📌 회장실
  #co-회장실 — Andrew 지시/공지

🛡️ 리스크챌린지
  #co-리스크 — 반대 의견/리뷰

🎬 콘텐츠 사일로
  #co-콘텐츠 — 아이디어/대본/반응 분석
  #co-바이럴 — 배포 트래킹/성과

🔨 프로덕트 사일로
  #co-프로덕트 — FRIDAY 작업 분배/스프린트
  #co-개발 — 에이전트간 개발 대화 (PR, 코드리뷰)
  #co-mybidwise — MyBidWise 전용

📈 투자 사일로
  #co-투자 — 브리핑/시그널/리포트
  #co-매매일지 — 매매 기록
  #co-리서치 — 매크로/뉴스/분석

⚙️ 인프라
  #co-인프라 — 배포/장애/모니터링
```

**총 10개 채널** (v1의 51개 → 80% 축소)

### 에이전트 대화 규칙
1. 에이전트끼리 해당 채널에서 직접 대화
2. 블로커/에스컬레이션 → #co-회장실에 태그
3. 일일 요약 → 각 사일로 총괄 채널에 자동 게시
4. Andrew는 아침에 #co-회장실만 확인하면 전체 파악 가능

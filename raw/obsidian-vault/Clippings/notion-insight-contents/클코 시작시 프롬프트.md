# 클코 시작시 프롬프트

Claude Code 프로젝트피드백루프 자동 설정 꿀팁
클코로 개발하는데,클코가 매번 프로젝트 구조 파악하느라시간 오지게 쓰지 않음?
직접 쓰면서 만든프로젝트 자동 분석 + 훅 설정 프롬프트공유하겠음
어떤 프로젝트에서든 명령어 한 줄이면- 프로젝트 구조 자동 파악- 맞춤형 훅 설정- 실수 방지 패턴 구축- 관련 파일 리마인더 설정
모든게 자동으로 세팅됨.
사용법 ⬇️1. 클코 켜고 아래 프롬프트 복붙2. 끝.

실제 작동 예시
세션 시작하면:
```<project-context>## Architecture OverviewCustomer → API Gateway → Services → DB...</project-context>```
이렇게 컨텍스트가 자동 로드됨.
파일 수정하면:
"📎 frontend/src/lib/types/ - FE TypeScript 타입도 확인?"
관련 파일 리마인더가 뜸.
같은 파일 여러번 고치면:
"📝 auth.py 3회 수정됨 - 실수 패턴이면 /add-mistake로 기록"
실수 패턴 감지 알림이 옴.

왜 이게 필요하냐면
Claude Code는 똑똑한데,매번 프로젝트 구조 탐색하는게- 시간 낭비- 컨텍스트 낭비- 같은 실수 반복
이거 한번 세팅해두면리소스 토큰량 60%는 줄어든다.진짜로.

ㅡ>

CLAUDE.md = 항상 알아야 할 정적 정보 (구조, 규칙, 패턴)훅 = 상황에 따른 동적 반응 (알림, 검증, 자동화)약간 model 건드리고나면 훅이 알아서 돌아서 schema도 확인하게 해주는 느낌이랄까?

# 프로젝트 자동 설정 (Auto Project Setup)
이 프로젝트를 분석하고 최적의 Claude Code 환경을 자동으로 구축해줘.
## 1단계: 프로젝트 분석- 프로젝트 타입 파악 (모노레포/풀스택/백엔드/프론트엔드)- 주요 기술 스택 확인- 디렉토리 구조 파악- 핵심 진입점 (엔드포인트, 페이지, 서비스) 식별
## 2단계: CLAUDE.md 생성/업데이트프로젝트 루트의 CLAUDE.md에 다음 내용 추가:- **Quick Overview**: 1-2줄 프로젝트 요약- **Core Flows**: 주요 기능별 핵심 파일 경로 테이블- **Development Commands**: 빌드, 테스트, 린트 명령어- **Permissions**: 허용/금지 작업 목록
## 3단계: 컨텍스트 캐시 생성`.claude/cache/project-context.md` 파일 생성:- Architecture Overview (ASCII 다이어그램)- Key Entry Points 테이블- Common Patterns (새 기능 추가 시 패턴)- Database Models 요약 (있으면)
## 4단계: 훅 설정`.claude/settings.json`에 다음 훅 추가:
### SessionStart Hook세션 시작 시 프로젝트 컨텍스트 자동 로드하는 쉘 스크립트:`.claude/hooks/load-project-context.sh`
### PostToolUse Hook (Edit/Write 후)1. **관련 파일 리마인더**: 특정 파일 수정 시 연관 파일 알림   - Model 수정 → Schema, FE 타입 확인 알림   - Router 수정 → API 클라이언트 확인 알림   - 프로젝트 특성에 맞게 패턴 정의
2. **실수 감지**: 같은 파일 3회 이상 수정 시 로그 + 알림   - `.claude/cache/mistake-candidates.jsonl`에 기록
## 5단계: 커스텀 명령어`.claude/commands/` 폴더에 유용한 명령어 생성:- `check.md`: 빠른 타입 체크 (프로젝트에 맞게)- `add-mistake.md`: 실수 패턴 수동 추가
## 최종 확인사항- [ ] CLAUDE.md에 Core Flows 섹션 있음- [ ] `.claude/cache/project-context.md` 생성됨- [ ] `.claude/settings.json`에 SessionStart, PostToolUse 훅 등록됨- [ ] 훅 스크립트에 실행 권한 부여 (chmod +x)
완료되면 설정된 내용 요약해서 알려줘.

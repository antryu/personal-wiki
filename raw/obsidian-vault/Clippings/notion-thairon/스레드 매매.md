# 스레드 매매

Threads Fund — 시스템 기능 명세서
1. 개요
Threads Fund는 자율 롱숏 헤지펀드 플랫폼입니다.
매크로 투자 논문(Macro Thesis)에 기반하여 포트폴리오를 관리하는 시스템으로, 단일 브로커리지 계좌에서 복수의 가상 전략 포트폴리오를 독립적으로 운영합니다.
핵심 목표:
  • 액티브 데이트레이딩이 아닌, 매크로 논문 기반 포지션 관리 및 리밸런싱
  • 전략별 독립 포트폴리오 운영 (가상 분리)
  • 투자 논문의 유효성을 지속적으로 검증하고 포지션에 반영
  • 완전한 감사 추적 (Audit Trail) 및 성과 추적
2. 시스템 아키텍처
  • Python 백엔드: hedgefund.py 엔진 + FastAPI 대시보드 서버
  • SQLite 데이터베이스: 20개 이상 테이블, 감사 추적, FIFO 로트 회계
  • 3가지 독립 모드: Mock (시뮬레이션) / IBKR Paper / IBKR Live
    — 각 모드별 완전히 별도의 데이터베이스 운영
  • IB Gateway Docker 컨테이너 + IBC 자동 로그인 (headless)
  • Nginx 리버스 프록시 + systemd 서비스
  • WebSocket 실시간 가격 피드
3. 핵심 기능
3.1 멀티 전략 포트폴리오
  • 가상 전략별 독립 allocation, cash, exposure 관리
  • 같은 종목이 여러 전략에 독립적으로 존재 가능
  • FIFO 로트 매칭 (세금 및 추적용)
  • 롱/숏 포지션 + 마진 트래킹
  • NAV 계산: cash + long MV − short MV + unassigned cash
3.2 투자 논문 엔진 (Thesis Engine)
  • 전략별 매크로 투자 논문 관리 (한국어 지원)
  • 자동 뉴스 스캔 (Brave Search + DuckDuckGo 듀얼 소스)
  • 논문 유효성 검토: strengthening / confirmed / weakening / invalidated
  • 프리마켓 리뷰 (시장 개장 30분 전 자동 실행)
  • 뉴스 감성 분석 + 포지션 변경 권고
  • 리뷰 히스토리 데이터베이스 저장
3.3 시그널 & 실행 엔진
  • 시그널 큐: BUY / SELL / SHORT / COVER / HOLD
  • 의사결정 엔진 (confidence, source, horizon 기반)
  • 주문 큐 + Paper/Live 실행
  • 시장 시간 체크 (강제 실행 옵션)
  • 시그널 소스별 점수 추적
3.4 리스크 관리
  • 포지션별 / 섹터별 / 전략별 exposure 모니터링
  • Gross/Net exposure 한도
  • 긴급 정지 (Kill Switch) + 재개
  • 포지션 reconciliation
  • 드로다운 모니터링
3.5 가격 피드
  • Yahoo Finance + Finnhub 실시간 가격
  • 서버사이드 캐싱 (15초 TTL, API 키 브라우저 노출 방지)
  • WebSocket 실시간 가격 푸시 (/ws/prices)
3.6 IB Gateway 연동
  • Interactive Brokers Gateway Docker 컨테이너
  • IBC 자동 로그인 (headless, Xvfb)
  • Paper/Live 모드 전환
  • 주문 실행 및 포지션 동기화
4. 대시보드
4.1 공개 대시보드
  • Bloomberg 스타일 다크 테마 UI
  • 실시간 가격 플래시 애니메이션
  • 전략별 필터 탭 (NAV, P/L, 포지션, exposure 필터링)
  • Exposure 바 (Long/Short/Net + 현금 분류: 총현금/숏담보/여유현금)
  • 모바일 반응형
  • Sharpe ratio (전략별 + 펀드 전체)
4.2 관리자 대시보드
  • API 키 인증
  • 시그널 제출
  • 리스크 컨트롤
  • 긴급 정지
  • 로트 뷰어
  • Reconciliation
5. 자동화
  • 뉴스 스캔: 평일 3회 (8am / 2pm / 8pm UTC) — Brave + DDG 듀얼 소스
  • 프리마켓 논문 리뷰: 평일 9:00 AM ET (개장 30분 전)
  • 스케줄러: 8개 예약 작업
  • Webhook API: 외부 시그널 수신 (API 키 인증)
6. 기술 스택
  • 언어: Python 3.12
  • DB: SQLite (WAL 모드, 20+ 테이블)
  • API: FastAPI + WebSocket
  • 프론트엔드: HTML/CSS/JS (바닐라, 프레임워크 없음)
  • 브로커: Interactive Brokers (ibapi)
  • 가격: Yahoo Finance, Finnhub
  • 검색: Brave Search API, DuckDuckGo
  • 인프라: Docker, Nginx, systemd, Xvfb
  • 테스트: 82개 테스트, 100% 통과
7. TODO — 개발 로드맵
운영 철학: Threads Fund는 액티브 데이트레이딩 펀드가 아닙니다.
매크로 투자 논문(Macro Thesis)에 기반하여 전략을 수립하고, 시간에 따른 포트폴리오 성과를 추적하며, 매크로 환경 변화에 따라 포지션을 조정하는 장기 매크로 펀드입니다.
Phase 1: 핵심 기능 완성 (Core)
  • IB Gateway Paper Trading 연동 완료 — 실제 주문 실행 파이프라인 구축
  • 실시간 가격 반영 → P/L 계산 정상화
  • Daily NAV 스냅샷 자동화 — Sharpe Ratio 계산, 장기 성과 추적 기반 마련
  • Mock DB 스케일 복구 — 시뮬레이션 환경 정상화
Phase 2: 매크로 투자 논문 시스템 (Thesis Engine)
  • 전략별 매크로 투자 논문 상세 관리 — 진입 근거, 핵심 지표, 리스크 요인 구조화
  • 논문 업데이트 히스토리 추적 — 언제, 왜 변경했는지 완전한 기록
  • 매크로 지표 모니터링 대시보드 — 금리, 인플레이션, 원자재, 환율 등 핵심 지표
  • 논문 기반 리밸런싱 트리거 — 매크로 조건 변화 시 자동 알림 발송
Phase 3: 포지션 관리 & 리밸런싱 (Position Management)
  • 전략별 목표 비중 설정 + 드리프트(Drift) 모니터링
  • 리밸런싱 제안 자동 생성 — 현재 비중 vs 목표 비중 비교 분석
  • 포지션 변경 시 근거 기록 필수 — 모든 변경에 대한 감사 추적(Audit Trail)
  • 세금 효율적 매도 순서 — Tax-Lot Optimization (FIFO 로트 기반)
Phase 4: 성과 분석 & 리포팅 (Analytics)
  • 전략별 / 펀드 전체 성과 차트 — 시계열 기반 시각화
  • 벤치마크 대비 성과 비교
  • 월간 / 분기 리포트 자동 생성
  • 드로다운 분석, 승률, 평균 수익/손실 등 핵심 성과 지표
Phase 5: 인텔리전스 (Intelligence)
  • 매크로 뉴스 모니터링 + 포트폴리오 영향도 분석
  • 어닝 캘린더 연동 — 보유 종목 실적 발표일 자동 추적
  • 섹터 로테이션 감지 — 자금 흐름 변화 포착
  • 상관관계 매트릭스 — 전략 간, 종목 간 상관계수 분석
Phase 6: 프로덕션 (Production)
  • IBKR Live 모드 연동 — 실계좌 거래 활성화
  • 자동 주문 실행 — 승인 기반 워크플로우 (사람 확인 후 실행)
  • 알림 시스템 — 텔레그램으로 포지션 변경, 리스크 경고, 매크로 이벤트 알림
  • 백업 & 복구 — 데이터베이스 자동 백업 및 재해 복구 절차

### 핵심: 에이전트 역할 분리
PM 에이전트가 첫 순서 → 기획서/PRD 작성 → 다른 에이전트에게 분배

### 6개 서브 에이전트:
1. **PM** — 프로젝트 매니저, UX 체크리스트, 에이전트 할당 규칙
2. **FRONTEND** — CSS 변수, 폰트, 페이지별 구현, SDK 연동, API 호출 패턴
3. **BACKEND** — API 엔드포인트, OAuth, 사용량 제한, 파싱 코드
4. **PROMENG** — 프롬프트 엔지니어, AI 활용 프롬프트 파일 작성
5. **QA** — P0~P3 심각도 분류, 개인정보 노출 검사, 위기 대응 테스트
6. **UXREVIEW** — UX/UI/SEO, 버튼 위치, 폰트 가독성, 유저 편의

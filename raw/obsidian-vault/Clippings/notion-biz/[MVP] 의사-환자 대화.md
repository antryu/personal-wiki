# [MVP] 의사-환자 대화

카테고리: 비즈니스
우선순위: 높음
진행상태: 완료

[https://doc-talk-insight-ai.lovable.app](https://doc-talk-insight-ai.lovable.app/)

[프롬프트 생성] 

병원에서 의사가 환자와 직접 대화한 내용을 데이터베이스화 해서 지속적으로 관리하는 사이트를 lovable.dev를 활용해서 구축하려고해. 우선 "개발 프로세스"를 제안해줘.

(개발 프로세스)

- 코딩을 모르는 초보개발자가 풀스택으로 웹서비스를 만들기 위한 개발 프로세스
- 사용하기에 효율적인 AI 프로그램 추천

**개발 프로세스 (lovable.dev 활용 기준)**

**목표:** 코딩을 모르는 초보개발자가 **의사-환자 대화 저장 및 분석 웹서비스**를 만들기 위한 단계별 가이드입니다.

### 🔧 개발 단계별 프로세스

| 단계 | 내용 | 추천 도구 |
| --- | --- | --- |
| 1단계 | 아이디어 정리 및 플로우 정의 | Notion, Whimsical |
| 2단계 | 프론트엔드 화면 설계 (UI/UX) | **lovable.dev** + Figma |
| 3단계 | 사용자 입력/버튼 로직 정의 | **lovable.dev**의 워크플로우 기능 |
| 4단계 | AI 음성인식 연동 | **Naver Clova Speech Recognition (CSR)** API |
| 5단계 | 실시간 텍스트 저장 및 표시 | **lovable.dev**의 리액션 + DB 연결 |
| 6단계 | 데이터 저장 / 불러오기 | **lovable.dev** 내장 DB 또는 Supabase 연동 |
| 7단계 | AI로 예상 병명 추론 | GPT API 연동 (lovable.dev에서 외부 API 호출 지원) |
| 8단계 | 배포 및 실사용 테스트 | lovable.dev 내 1클릭 배포 기능 활용 |

[1단계] 아이디어 정리 및 플로우 정의

- 아이디어(MVP)
    - 의사 진료 시, 환자와의 대화를 기록하고 텍스트로 요약 분석해주는 시스템
    - 증상을 바탕으로 가능성 높은 병명 제공
    - 나중에 다시 환자가 올때 과거 이력 및 자료를 참고할 수 있어야 함
    - 향후 추가 기능
        - 대화 내용으로 부터 “의사법”과 관련한 정보 제공
    - 필요 AI: Lovable, Naver Clova, ChatGPT
- 플로우
    - (의사) 회원가입
    - 환자와 의사의 대화를 음성으로 저장하고, 실시간 텍스트화
    - 대화 종료 시 DB에 저장
    - AI가 대화를 분석해 예상 병명 3개 제시
    - 환자 재방문 시 기존 기록 조회 가능

[2단계] 프론트엔트 화면 설계

아래 "진료프로세스" 내용을 포함하는 lovable.dev 에 넣을 홈페이지 프롬프트를 만들어줘. 

“진료 프로세스”

(1) 의사가 환자의 '이름', '나이' 를 입력
(2) 의사가 환자에게 '대화저장에 대한 동의여부' 확인 및 체크박스에 표시
(3) 의사가 "대화저장" 버튼을 누르면 환자와 의사가 나눈 대화를 텍스트로 실시간으로 창에 표시
- 네이버 클로바와 연동
(4) 대화가 끝난 후 "대화종료" 버튼을 누르면 대화가 데이터베이스에 저장
(5) "예상병명" 버튼을 누르면 대화에 기반해서 예상 병명을 3개 제공

(6) 동일한 환자재방문 시, 환자 이름 입력
- 이름 입력 시, 과거 대화 및 예상 병명을 표시

(7) 배경화면은 병원의사가 사용하는 느낌으로 고급진 색깔로 적용해줘.

음성 인식 결과가 추가될 때까지 확실하게 기다려줘

최근 환자 기록 표시와 예상 병명 분석 기능을 추가하겠습니다.

대화종료 시 음성 처리가 완료되기 전에 너무 빨리 종료되는 문제가 있습니다. 음성 인식 결과가 비어있어서 대화 기록이 저장되지 않습니다.

대화 중에 "진료 종료"를 누르면 음성 처리 완료를 기다린 후 모든 대화 기록과 함께 진료완료창으로 이동됩니다.

진료대화 창을 독립적인 컴포넌트로 분리하겠습니다.

# 거래 로직 상세 (진입 → 청산)
*기준: BBStrategy.backtest() + 레짐 필터 활성화*
*소스: `src/strategy.py` line 1030~1400*

## 전체 흐름도

```
30분봉 데이터 수신
    ↓
[1] 레짐 판단 (MA20/MA200 + ATR)
    ↓
[2] BB(40,2) 신호 생성 + 필터링
    ↓
[3] 진입 조건 충족 → 포지션 진입
    ↓
[4] 매 봉마다 청산 조건 체크
    ↓
[5] 청산 조건 충족 → 포지션 청산
    ↓
거래 기록 저장 → 다음 신호 대기
```

---

## [1] 레짐 판단

**매 30분봉마다 계산** (`calculate_regime()`, line 482)

| 조건 | 레짐 | 코드 | 허용 방향 |
|------|------|------|----------|
| MA20 > MA200 & ATR < ATR_AVG×1.3 | UP_LOW | 1 | Long only |
| MA20 > MA200 & ATR ≥ ATR_AVG×1.3 | UP_HIGH | 2 | Long only |
| MA20 ≤ MA200 & ATR < ATR_AVG×1.3 | DOWN_LOW | 3 | Short only |
| MA20 ≤ MA200 & ATR ≥ ATR_AVG×1.3 | DOWN_HIGH | 4 | Long(하단) + Short(상단) |

- MA20 = SMA(close, 20)
- MA200 = SMA(close, 200)
- ATR = EMA(TR, 20)
- ATR_AVG = SMA(ATR, 20)

## [2] 신호 생성

**BB(40,2) 이탈 감지** (`generate_signals()`, line 600~)

### 진입 신호 조건
```
LONG 진입:
  close < bb_lower (BB 하단 이탈)
  AND 레짐이 Long 허용 (UP_LOW, UP_HIGH, DOWN_HIGH)
  AND 경제지표 블랙아웃 아님 (선택)

SHORT 진입:
  close > bb_upper (BB 상단 이탈)
  AND 레짐이 Short 허용 (DOWN_LOW, DOWN_HIGH)
  AND 경제지표 블랙아웃 아님 (선택)
```

### 필터 (레짐 모드에서 비활성)
- RSI 필터: **OFF** (레짐 사용 시 자동 비활성, line 659-660)
- Trend 필터: **OFF** (레짐 사용 시 자동 비활성)
- 밴드폭 필터: 선택적
- 거래량 필터: 선택적
- 시간대 필터: 선택적

## [3] 진입

### 진입 시점
- `signal_close` (기본): 신호봉 종가에 진입
- `next_open`: 신호 발생 후 다음 봉 시가에 진입

### 진입 시 설정값 계산

```
1. 진입가 = close (또는 다음봉 open)
2. 레짐 코드 저장 (entry_regime)
3. 손절가 계산:
   - 레짐 OFF: ATR 기반 or 고정 손절
   - 레짐 ON: 레짐별 손절 (pt → 가격 변환: /4)
     LONG: stop_loss = entry_price - (regime_SL / 4)
     SHORT: stop_loss = entry_price + (regime_SL / 4)
4. 익절가 계산: BB 반대편 or 고정값
5. 트레일링 스탑 = None (아직 미활성)
```

### 레짐별 손절 (포인트)

| 레짐 | SL(pt) | 가격 변환 |
|------|--------|----------|
| UP_LOW | 15pt | 15/4 = 3.75 |
| UP_HIGH | 25pt | 25/4 = 6.25 |
| DOWN_LOW | 15pt | 15/4 = 3.75 |
| DOWN_HIGH | 25pt | 25/4 = 6.25 |

## [4] 포지션 보유 중 (매 봉마다)

### 4-1. 극값 업데이트
```python
max_price = max(max_price, row['high'])   # 보유 중 최고가
min_price = min(min_price, row['low'])    # 보유 중 최저가
```

### 4-2. ATR 트레일링 손절 업데이트 (ATR 모드 시)
```
LONG: 고점 갱신 시 → 손절가 상향 (유리한 방향만)
SHORT: 저점 갱신 시 → 손절가 하향 (유리한 방향만)
```

### 4-3. 레짐 트레일링 스탑 업데이트
```
레짐 모드: use_regime_filter=True → 자동으로 트레일링 활성화

LONG:
  profit = max_price - entry_price
  if profit >= trailing_start:
    trailing_stop = max(trailing_stop, max_price - trailing_step)

SHORT:
  profit = entry_price - min_price
  if profit >= trailing_start:
    trailing_stop = min(trailing_stop, min_price + trailing_step)
```

### 레짐별 트레일링 파라미터 (포인트)

| 레짐 | Trail Start | Trail Step | 가격 변환 |
|------|------------|------------|----------|
| UP_LOW | 30pt | 5pt | 7.5 / 1.25 |
| UP_HIGH | 50pt | 7pt | 12.5 / 1.75 |
| DOWN_LOW | 30pt | 5pt | 7.5 / 1.25 |
| DOWN_HIGH | 40pt | 7pt | 10.0 / 1.75 |

## [5] 청산 조건 (우선순위 순)

### 청산 가격 기준표

| # | 청산 유형 | LONG 조건 | SHORT 조건 | 청산 가격 |
|---|----------|-----------|-----------|----------|
| 1 | **손절** | Low ≤ stop_loss | High ≥ stop_loss | stop_loss (지정가) |
| 2 | **트레일링** | Low ≤ trailing_stop | High ≥ trailing_stop | trailing_stop |
| 3 | **익절** | High ≥ take_profit | Low ≤ take_profit | take_profit |
| 4 | **BB 재진입** | Close > bb_lower | Close < bb_upper | Close (종가) |
| 5 | **MA 청산** | Close < MA | Close > MA | Close (종가) |
| 6 | **시간 청산** | bars ≥ max_bars | bars ≥ max_bars | Close (종가) |
| 7 | **장종료** | 장종료 시간 도달 | 장종료 시간 도달 | Close (종가) |
| 8 | **경제지표** | 발표 N시간 전 | 발표 N시간 전 | Close (종가) |

### 동시 도달 규칙
> 같은 봉에서 손절/익절 동시 도달 시 → **손절 우선** (보수적 가정)
> 30분봉 내 High/Low 도달 순서를 알 수 없으므로 최악 가정

### 장종료 시간
- 썸머타임 (3월~11월): 06:00 KST (전일 기준)
- 윈터타임 (11월~3월): 07:00 KST (전일 기준)
- `get_session_close_time()` 함수로 자동 판별

## 실제 예시 (LONG + UP_LOW 레짐)

```
[봉 100] BB 하단 이탈 감지 (close < bb_lower)
         레짐 = UP_LOW (MA20 > MA200, 저변동)
         → LONG 진입: entry_price = 6000.00
         → stop_loss = 6000 - (15/4) = 5996.25
         → trailing_start = 30/4 = 7.50
         → trailing_step = 5/4 = 1.25

[봉 101] high=6005, low=5998
         profit = 6005 - 6000 = 5.00 (< 7.50)
         → 트레일링 미활성

[봉 105] high=6010, low=6003
         max_price = 6010
         profit = 6010 - 6000 = 10.00 (≥ 7.50)
         → trailing_stop = 6010 - 1.25 = 6008.75

[봉 110] high=6015, low=6008
         max_price = 6015
         profit = 6015 - 6000 = 15.00
         → trailing_stop = 6015 - 1.25 = 6013.75

[봉 112] high=6014, low=6012
         low(6012) ≤ trailing_stop(6013.75)
         → 트레일링 스탑 청산: exit_price = 6013.75
         → PnL = 6013.75 - 6000.00 = +13.75 (55pt)
```

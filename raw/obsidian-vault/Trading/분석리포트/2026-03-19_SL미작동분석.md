- **진입가:** 6813.75

- **청산 사유:** ATR_S (트레일링/ATR 청산)

## 원인 분석

### 1. 서버 스탑주문 없음 (근본 원인)
- BuyFX/SellFX는 시장가 주문만 발송 (nHogaGb="1")
- 키움 서버에 STOP 주문(nHogaGb="3") 안 걸림
- SL 체크가 전부 엑셀 VBA 내부에서만 동작
- **엑셀 멈추면 SL 무력화**

### 2. testloop 35틱 딜레이
- `OnReceiveRealData`에서 `testloop < 35` → 35번째 틱에서만 로직 실행
- 나머지 34틱은 스킵 → SL 체크 지연
- 천천히 내려가는 장에서도 체크 빈도 부족

### 3. ATR 동적 SL 계산
```vb
atrSL = DC25 * 12   ' ATR_AVG × 12
If atrSL < 10 Then atrSL = 10   ' 최소 10틱 (2.5pt)
If atrSL > 60 Then atrSL = 60   ' 최대 60틱 (15pt)
```
- 실제 SL 값 확인 필요: 엑셀 DF25 셀
- 옵시디언 파라미터 문서(레짐별 15/25pt)와 VBA 실제 코드(ATR 동적)가 다름

### 4. 진입 자체 문제
- 진입가 6813.75 → 최고가 6810.25 (진입부터 마이너스)
- RSI 19.26, ADX 19.42 = 과매도 + 추세 약함
- 급락 아님, 5시간 동안 서서히 하락

### 5. flagCut/flagTrend 의존성
- SL 체크 조건: `If flagCut = 11 And flagTrend = 2`
- 진입 시 설정되지만 중간에 초기화될 가능성 있음
- 플래그 기반 로직이 너무 복잡 → 빈틈 발생

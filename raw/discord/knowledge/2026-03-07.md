💡 핵심 인사이트 5가지

**1. 마이크로 툴의 시대**

예전엔 "이 디버그 툴 만드는 데 일주일이겠네" → 포기. 이젠 에이전트한테 5분이면 만듦.

**M1Pro16Gbot** (PM 12:57)
• 경계 편집기, 물 분류기, 학습 데이터 생성기, 색상 보정 도구 등 수십 개를 즉석 제작
• 패턴: **CLI tool → Library → Application** (점진적 추상화)

**2. 이미지 모델은 아직 멀었다**

코드 에이전트는 에러 → 스택트레이스 → 자기 수정 루프가 있다. 이미지 모델은 **자기 실수를 못 본다.**

• Gemini 3 Pro조차 "여기 이음새가 있다", "이 나무 텍스처가 틀렸다"를 안정적으로 감지 못함
• 자동 QA 포기 → 수동 리뷰가 전체 작업의 90%

**3. 편집 문제 (The Edit Problem)**

코드는 특정 줄을 가리킬 수 있다. 이미지는 "저 나무"를 가리킬 방법이 없다.

• 코드: 버그 수정 = 파일 수정. 이미지: 나무 수정 = 전체 다시 생성 (diffusion의 한계)
• 마스킹, 투명도, few-shot 프롬프팅 → 전부 불가능

**4. 소프트웨어 엔지니어링은 사라지지 않는다 — 추상화 레벨만 올라간다**

"생성 큐의 도메인 로직은 내가 스펙을 짰다. 구현 코드는 한 번도 안 봤다. 진심이다."

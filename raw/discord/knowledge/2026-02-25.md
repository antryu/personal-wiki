Steerling 심층 분석

1. 핵심 문제의식

기존 LLM은 블랙박스 → post-hoc interpretability(SAE 등)로 사후 분석. GuideLabs는 **설계 단계부터 해석 가능성을 내장**하자는 접근.

2. 아키텍처: Concept Module

트랜스포머 backbone과 LM head 사이에 **concept bottleneck** 삽입:

```
hidden state h
  → f(h): known concept 활성화 (33K개, 시그모이드)
  → g(h): unknown concept 활성화 (101K개)
  → known embeddings + unknown embeddings = h̄
  → LM head(h̄) → logits
```

**M1Pro16Gbot** (PM 03:30)
**핵심:** h가 직접 LM head로 가는 경로를 **차단**. 모든 예측이 반드시 concept을 통과.

3. 학습: 4개 Loss

• **L_token**: 일반 LM loss (masked/next-token), h̄ 기반
• **L_concept**: chunk-level concept 라벨과 매칭 (ATLAS로 자동 라벨링)
• **L_indep**: known/unknown 간 cross-covariance 최소화 (중복 방지)
• **L_rec**: unknown head가 `h - known_embedding` 잔차를 복원

4. Causal Diffusion (CDLM) — 새로운 생성 방식

```
|                  | AR      | Full Diffusion     | Block Diffusion           | Causal Diffusion  |
| ---------------- | ------- | ------------------ | ------------------------- | ----------------- |
| Attention        | causal  | full bidirectional | block-causal + clean half | block-causal only |
| 생성 단위            | 1 token | multi-token        | multi-token               | multi-token       |
```

**M1Pro16Gbot** (PM 03:30)
```
| 학습 비용            | 기준      | 기준                 | 2x (clean half 복제)        | 1x                |
| Scaling exponent | -0.074  | -0.038 (나쁨)        | -                         | -0.072 (AR급)      |
```
**Block-causal**: 64토큰 블록 내 bidirectional, 블록 간 causal. Block Diffusion의 clean half 제거 → 학습 비용 절반으로 AR급 스케일링 달성.

5. Attribution — 3단계 추적 체인

1. **Concept → Token**: logit이 concept activation의 선형 합 → 정확한 기여도 분해 (근사 아님)
2. **Input → Token**: Integrated Gradients (MASK를 baseline으로)
3. **Input → Concept**: 어떤 입력 토큰이 어떤 concept을 활성화했는지

**완전한 체인:** `입력 토큰 → concept 활성화 → 출력 토큰`

concept module 추가해도 스케일링 법칙 거의 동일. 모델 커질수록 overhead 비율은 줄어듦. HellaSwag/ARC/PIQA 등에서 base 모델과 concept 모델 성능 차이 미미.

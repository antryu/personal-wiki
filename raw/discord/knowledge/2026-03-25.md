아니, **Vercel 자체를 대체하는 건 아냐.**

**Vercel = 실제 배포/호스팅 플랫폼** (프로덕션)
**Emulate = Vercel의 API를 로컬에서 흉내** (개발/테스트)

비유하면:
- Vercel = 실제 은행
- Emulate = 은행 ATM 시뮬레이터 (개발자가 테스트용으로 쓰는 것)

**구체적으로:**
- 프로덕션 배포 → 여전히 Vercel 필요
- 근데 **테스트 코드**에서 "Vercel API로 프로젝트 목록 가져오기" 같은 걸 테스트할 때 → 진짜 Vercel API 안 쳐도 됨
- CI에서 GitHub Actions 돌릴 때 → GitHub API 호출도 로컬로 처리

**_y/MyBidWise 기준으로:**
- `vercel deploy` → 여전히 진짜 Vercel
- `npm test` → emulate로 외부 API 의존성 제거 (오프라인에서도 테스트 통과)

🔧 기술 스택 차이

**ChromaDB**

• **역할:** 벡터 유사도 검색
• **장점:** 로컬 실행, 오픈소스
• **단점:**
  • 파일 기반 (확장성 낮음)
  • Vercel에서 실행 어려움 (stateless)
  • 별도 서버 필요 or 배포 시 번들링

**Supabase PostgreSQL**

**M1Pro16Gbot** (PM 01:26)
• **역할:** 법령 원문 저장, 메타데이터 관리
• **장점:** 관리형 PostgreSQL, 무료 티어
• **단점:**
  • API 키 관리 필요
  • 500MB 제한 (무료)
  • 데이터 직접 수집/관리

**법망 pgvector**

• **역할:** ChromaDB + Supabase 역할 통합
• **장점:**
  • 단일 API 호출로 완결
  • 20만 조문 이미 임베딩 완료
  • PostgreSQL 기반 (안정성)
• **단점:**
  • 커스터마이징 불가 (법망 데이터만)

**M1Pro16Gbot** (PM 03:47)
📋 LeCun/JEPA에 대한 5가지 비판

**1. 데이터: "정보 밀도가 중요하지, 볼륨이 아니다"**

• LeCun: "4살 아이도 1.1^14 bytes 시각 데이터를 처리한다" → 센서리 입력이 텍스트보다 우월
• Xing 반박: 비디오 데이터는 **중복이 극심**. 언어는 수천 년간 인간 경험을 **압축**한 것. "정의", "후회" 같은 개념은 영상에 없고 텍스트에만 있음
• **결론**: 모든 모달리티를 써야 함. 비디오만으로는 부족

**2. 표현: "연속 vs 이산? 둘 다 필요"**

• LeCun: 이산 토큰 쓰지 마. 연속 임베딩만 써야 gradient 기반 최적화가 됨
• Xing 반박: 인간 인지는 **연속 신호를 이산 개념으로 범주화**하여 추론함. VQ-VAE로 이산 토큰도 충분히 fine-grained 표현 가능 (Theorem 1로 수학적 증명)
• **결론**: 연속 + 이산 혼합 표현이 최적

**3. 아키텍처: "Autoregressive가 적이 아니다"**

• LeCun: AR 모델은 에러가 누적돼서 망함 → JEPA(encoder-encoder) 써야 함

**M1Pro16Gbot** (PM 03:47)
• Xing 반박: **JEPA도 사실 autoregressive임!** `s' = f(s, a)`를 반복 적용하면 결국 동일한 순차 생성. 다만 JEPA는 **확률적 불확실성을 모델링하지 않는 퇴화된 분포**(Dirac delta). 카오스 시스템에서 에러 누적은 모델 구조 문제가 아니라 시스템 본질
• **대안**: GLP (Generative Latent Prediction) — 계층별 다른 granularity로 예측
  • 하위: 디퓨전 모델 (연속 perceptual)
  • 중위: VQ-VAE + AR Transformer (이산 토큰)
  • 상위: LLM (언어 추론, 장기 계획)

**4. 목적 함수: "Latent space에서만 학습하면 collapse한다" ⚠️**

• LeCun: 픽셀 복원할 필요 없음. latent space에서 L2 loss만으로 충분
• Xing 반박 (핵심!):
  • **Proposition 1**: latent loss만 쓰면 **collapse**됨. encoder가 모든 입력을 상수 c로 매핑하면 loss=0 → global minimum. "아무것도 안 배우고 만점"
  • **Proposition 2**: generative loss (decoder 있으면) collapse가 불가능 — 다른 observation에 대해 다른 출력 만들어야 하니까
  • **Theorem 2**: L_latent ≤ L_gen + ε — latent loss를 줄여도 실제 관측과 일치한다는 보장 없음
• JEPA의 SIGReg (variance + covariance 정규화)는 이 collapse를 방지하기 위한 **휴리스틱**이고, 수학적으로 깔끔하지 않음
• **결론**: 관측 데이터로 anchoring하는 generative loss가 더 안정적

**5. 사용법: "MPC가 아니라 RL이 맞다"**

**M1Pro16Gbot** (PM 03:47)
• LeCun: RL은 trial이 너무 많이 필요. MPC가 나음
• Xing 반박: MPC는 짧은 수평선에서만 최적이고 복잡한 장기 목표에 취약. RL은 WM이 생성한 시뮬레이션 경험으로 학습 가능 (DreamerV3 식)
• **결론**: WM으로 경험 시뮬레이션 → RL agent 학습이 더 범용적

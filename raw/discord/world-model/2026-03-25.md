## Step 5 상세: Predictor가 어떻게 예측하는가

---

### 입력

Predictor에 들어가는 것 2가지:

```
1. 보이는 조각들의 latent vectors (10%의 인코딩 결과)
2. 마스크 토큰 (가려진 위치를 알려주는 빈 토큰)
```

### 구체적으로

16개 tubelet 중 2개만 보인다고 하자:

**M1Pro16Gbot** (AM 09:07)
```
위치3: latent_3 (인코딩된 실제 값)
위치10: latent_10 (인코딩된 실제 값)
나머지 14개: [MASK] + 위치정보
```

**마스크 토큰** = "여기에 뭔가 있긴 한데 뭔지 모른다" → 빈 벡터 + positional encoding(위치정보)

### Predictor 내부 (Transformer)

```
입력: [latent_3, latent_10, MASK_1, MASK_2, ... MASK_14]
      각각에 위치정보(어디에 있는 조각인지) 붙어있음
          ↓
      Transformer Self-Attention
          ↓
```

**M1Pro16Gbot** (AM 09:07)
```
      모든 토큰이 서로를 참조
          ↓
출력: [pred_1, pred_2, ... pred_14]  ← 가려진 14개의 예측 latent
```

### Self-Attention이 하는 일

MASK_1이 생각하는 과정:

> "나는 위치 1에 있다. 위치 3에 이런 정보가 있고, 위치 10에 저런 정보가 있다. 나는 위치 1이니까... 아마 이런 값이겠지."

**비유:** 퍼즐 2조각만 보고 나머지 14조각 맞추기. 근데 어떤 퍼즐인지(비디오의 맥락) 수백만 번 봐서 패턴을 알고 있음.

### 핵심: 왜 이게 작동하는가

비디오에는 **규칙성**이 있어:

**M1Pro16Gbot** (AM 09:07)
- 공이 왼쪽에서 오른쪽으로 가고 있으면 → 다음 프레임도 오른쪽
- 컵이 테이블 위에 있으면 → 다음 프레임에도 테이블 위
- 사람이 걷고 있으면 → 다리가 특정 패턴으로 움직임

Transformer가 이 규칙성을 **100만 시간** 보면서 학습. 그래서 10%만 봐도 나머지를 "상식적으로" 채울 수 있음.

"지금 상태 + 가격 변화가 이러면 → 다음 상태는?"

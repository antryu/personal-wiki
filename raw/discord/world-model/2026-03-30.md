**SIGReg = Signal Regularization**

JEPA가 collapse하지 않도록 **강제로 밀어주는 정규화**.

**2가지 loss:**

**1. Variance Loss (분산 손실):**
```
각 latent 차원의 분산 ≥ 1
```
- 모든 192 차원이 **활성화**되도록 강제
- "차원 0이 항상 0.5면 쓸모없음" → 분산 낮음 → penalty

**2. Covariance Loss (공분산 손실):**
```
차원 간 상관관계 → 0
```

**M1Pro16Gbot** (PM 10:52)
- 차원들이 **독립적**이도록 강제
- "차원 5 = 차원 10 * 2" 같은 중복 방지

**수식:**
```
SIGReg = var_loss + cov_loss

var_loss = max(0, 1 - var(z_i))  각 차원 i
cov_loss = |cov(z_i, z_j)|  i≠j
```

**목적:**
- JEPA는 decoder 없으니까 latent가 collapse할 위험 있음
- SIGReg이 **억지로** 모든 차원 쓰게 만듦
- Xing의 비판: "이건 휴리스틱이지, 수학적으로 깔끔한 해법 아님"

**네 JEPA 결과:**

**M1Pro16Gbot** (PM 10:52)
- 192/192 dim active → SIGReg이 **제대로 작동**했다는 증거
- Xing은 "이게 항상 작동한다는 보장 없음"이라고 주장

**GLP 차이:**
- GLP는 decoder 있으니까 SIGReg **없이도** collapse 안 함
- 더 자연스러운 방식

그래서 실험하는 거야 — SIGReg(휴리스틱) vs Decoder(자연스러운 방법) 뭐가 더 나은지.

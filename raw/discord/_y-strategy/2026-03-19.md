**M1Pro16Gbot** (PM 03:49)
에이전트 구조에서 flat vs hierarchy:

**Flat (CrewAI, AutoGen 등)**
```
Agent A ←→ Agent B ←→ Agent C
     (모두 동등, 직접 소통)
```

**Hierarchy (_y)**
```
Chairman (you)
  └─ Counsely (synthesis)
       ├─ Marketing team
       ├─ Engineering team
       ├─ Risk (independent)
       └─ Finance team
```

**M1Pro16Gbot** (PM 03:49)
**Hierarchy가 나은 이유:**

**1. 정보 과부하 방지**
- Flat: 30개 에이전트가 서로 다 대화 = 조합 폭발 (30×29 = 870 연결)
- Hierarchy: 각 팀 내부만 소통 → 위로 요약 → 관리 가능

**2. 독립성 보장**
- Flat: 한 에이전트 의견이 나오면 나머지가 앵커링
- Hierarchy: 부서별 격리 → 합성 단계 전까지 서로 안 봄

**3. 책임 추적**
- Flat: "누가 이 결론 냈어?" → 모름
- Hierarchy: 리포트에 부서 + 에이전트 이름 + 근거 명확

**4. 확장성**
- Flat: 에이전트 5개 넘으면 카오스
- Hierarchy: 부서 추가만 하면 됨 (기존 구조 안 건드림)

**M1Pro16Gbot** (PM 03:49)
**5. 인간이 이미 검증한 구조**
- 100년간 기업이 쓰는 구조 = hierarchy
- 이유가 있어서 flat 조직이 아니라 위계가 존재하는 것

**한 줄 요약:**
> Flat은 에이전트 5개까지. 그 이상은 hierarchy 없으면 noise만 늘어남.

# S&P500 MES 자동매매 전략 분석 및 개선

> S&P500 MES(Micro E-mini) 자동매매 시스템의 실전 성과 분석, 전략 파라미터 최적화, 디스코드 웹훅 체결 알림 자동화 구축 과정을 기록한 문서. 2월 Long only 전략에서 3월 양방향 스캘핑으로 전환하며 승률이 극적으로 개선되었고, VBA-디스코드 연동을 통한 실시간 모니터링 파이프라인을 구축했다.
> Last updated: 2026-04-05

## Overview

2026년 2월부터 키움증권 HTS + VBA 매크로 기반으로 S&P500 MES(Micro E-mini S&P500) 자동매매를 운용 중이다. 2월 Long only 전략은 승률 ~35%, PnL -32.8pt로 부진했으나, 2/28 트레일링 파라미터 축소(Trail 30~50→15, Step 5~7→5) + 3/2 양방향(Long+Short) 전환 이후 3월 성과는 8전 8승 +25pt로 극적 개선되었다. 핵심 발견은 Trail Start 축소가 Short 전략을 살렸다는 점이며, 6년 백테스트에서도 현재 T15/5 설정이 최적(Sharpe 13.66)임이 확인되었다. 다만 신호사용료 $250/월 대비 순수익이 부족한 구조적 문제가 남아있다.

## Key Topics

### 실전 성과 분석 (3/2~3/12)

3월 실전 거래 데이터(키움증권 체결내역 캡쳐)를 정밀 분석한 결과:

- **총 16건 체결 = 8 라운드트립** (양방향: Long 3건 + Short 5건)
- **승률 100%**, PnL **+25.0pt ($125)**
- Short 성과: +14.75pt (5건 전승) — 2월 Short 0승 3패(-16pt)에서 완전 반전
- Long 성과: +10.25pt (3건 전승)
- 평균 보유시간 ~7분 (스캘핑)
- 신호비+커미션 포함 순수익: **~$32/10일** (월간 ~$3 수준)

### 전략 파라미터 최적화

**Trail Start/Step 변경이 핵심 전환점:**

| 시점 | Trail Start | Trail Step | 결과 |
|------|------------|------------|------|
| ~2/28 이전 | 30~50틱 (7.5~12.5pt) | 5~7틱 (1.25~1.75pt) | 승률 ~35%, Short 전패 |
| 2/28~ 현재 | 15틱 (3.75pt) | 5틱 (1.25pt) | 승률 100%, Short 전승 |

**6년 백테스트 검증 결과 — T15/5가 압도적 최적:**

| 설정 | 거래수 | 승률 | PnL | Sharpe | 월순익 |
|------|--------|------|------|--------|--------|
| T15/5 (현재) | 1,662 | 65.5% | +5,663pt | 13.66 | +$143 |
| T30/10 | 1,620 | 48.3% | +2,999pt | 7.02 | -$42 |
| T60/20 | 1,620 | 39.6% | +413pt | 1.03 | -$221 |

Trail을 넓히면 트레일링 발동 전에 손절에 먼저 걸리는 비율이 급증. **이 전략의 DNA는 빠른 소폭 이익 확보.** 익절폭 확대는 오히려 성과를 악화시킨다.

### DailyTradeCount 버그 발견 및 수정

하루 1회 매매 제한(`MAX_DAILY_TRADES = 1`)이 작동하지 않는 버그 발견:

**원인:** 카운트 증가 로직(`DailyTradeCount + 1`)이 청산 섹션(Line 1264)에 위치 — 진입 시점에는 카운트가 아직 0이라 조건 체크를 통과해버림.

**수정:**
- Line 1264~1266 (청산부 카운트 로직) 삭제
- 진입 8곳(`Call BuyFX` / `Call SellFX` 직전)에 `DailyTradeCount = DailyTradeCount + 1` 추가

### 디스코드 웹훅 체결 알림 구축

VBA → 디스코드 웹훅 자동 체결 알림 시스템을 구축:

**구조:**
1. `DiscordWebhook` 모듈 (SendEntryAlert, SendExitAlert, PostToDiscord)
2. `OnReceiveChejanData` 이벤트에서 체결 확인 시 자동 전송
3. 웹훅 URL: #snp500-trading 채널 연동

**수정 포인트:**
- 진입 체결: `OnReceiveChejanData` Line 215 부근, 체결가격 세팅 직후 6줄 추가
- 청산 체결: Line 242 부근, `N9 = "NaN"` 직전 7줄 추가

**이슈 해결 과정:**
- `Private Const`를 Sheet1 맨 아래에 넣어 "End Sub 다음에는 주석만" 에러 → Sheet1 맨 위로 이동
- 한글 깨짐 (`??`로 표시) → VBA ANSI(CP949) 인코딩 문제 → `ADODB.Stream` UTF-8 변환 또는 영어 메시지로 대체

**3/13 첫 실전 알림 성공:** SHORT 6660.00 진입 → 6657.00 청산 (+3.25pt)

### 자동 모니터링 파이프라인 구상

체결 알림 자동화를 넘어 전체 트레이딩 모니터링 파이프라인 구상:

- **체결 자동 수집:** 키움 VBA → 디스코드 웹훅 → DB 자동 적재
- **일간/주간 자동 분석:** 크론 기반 성과 리포트, 파라미터 재최적화
- **이상 감지:** 실전-백테스트 괴리 감지, 연속 손실 알림, VIX 고조 경고
- **자동 판단 트리거:** 승률/연패/VIX 임계치 기반 알림

## Key Decisions

- **2/28**: Trail Start 30~50→15틱, Trail Step 5~7→5틱으로 축소 — 백테스트 PnL +44% 개선 근거
- **3/2**: Short 전략 복귀 (양방향 전환) — Trail 축소 효과로 Short 생존 가능 판단
- **3/12**: 익절폭(TP) 확대 불필요 판단 — 6년 백테스트에서 T15/5가 최적(Sharpe 13.66) 확인
- **3/12**: 디스코드 웹훅 방식 채택 (텔레그램 대비 기존 인프라 활용 용이)
- **3/12**: DailyTradeCount 로직을 청산부→진입부로 이동 결정

## Results & Outcomes

- 2월 Long only → 3월 양방향 전환으로 승률 ~35% → 100% 개선
- Trail 15/5 설정이 6년 데이터 기준 최적임을 백테스트로 확인
- **수익성 문제 미해결:** 월 순수익 ~$3 (신호비 $250 대비 부족)

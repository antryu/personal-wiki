# ABAP AI Development Bridge

> This repository bridges SAP ABAP Development Tools (ADT) with the Model Control Protocol (MCP), enabling AI assistants like Claude to interact with ABAP codebases. It unlocks AI-agentic development, allowing natural language or DSL-based automation for reading, writing, debugging, and deploying ABAP code. Notable for its integration of AI-driven analysis and refactoring capabilities.
> Last updated: 2026-04-17

## Overview

**ABAP AI Development Bridge**, known as **Vibing Steampunk (vsp)**, is a Go-based tool designed to connect SAP ABAP Development Tools (ADT) with Model Control Protocol (MCP). This integration allows AI assistants such as Claude to access and manipulate ABAP codebases directly, enabling a new level of AI-assisted development in SAP environments.

The bridge supports reading and writing code, deploying changes, running tests, and analyzing ABAP packages through natural language or domain-specific languages. It is especially valuable for developers working with SAP systems like ECC and S/4HANA, and it enhances productivity through AI-powered automation and analysis.

Key features include package health analysis, directional boundary detection, LUW (Logical Unit of Work) classification, and transport history tracking. The project is open-source under the MIT License and hosted on GitHub with active development and community contributions.

## Key Points

- Bridges ADT and MCP to enable AI-assisted ABAP development.
- Offers package health analysis, boundary detection, and LUW classification.
- Supports multiple output formats for analysis reports: text, JSON, Markdown, HTML, Mermaid, and more.
- Detects side effects and transactional patterns in ABAP code.
- Facilitates intelligent refactoring, transport history tracking, and co-change analysis.

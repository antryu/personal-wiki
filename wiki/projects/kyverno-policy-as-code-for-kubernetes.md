# Kyverno: Policy-as-Code for Kubernetes

> Kyverno is a Kubernetes-native policy engine that enables security, compliance, automation, and governance through policy-as-code. It integrates with existing Kubernetes tools and supports supply chain security, making it a notable solution for platform engineering teams.  
> Last updated: 2026-04-14

## Overview

Kyverno is a unified policy-as-code tool designed for Kubernetes environments. It allows teams to enforce policies during resource creation, background scans, and container image verification. The project emphasizes seamless integration with tools like `kubectl`, `kustomize`, and Git, avoiding the need for new languages or workflows.

Kyverno supports validation, mutation, generation, and cleanup of Kubernetes resources. It also verifies container image signatures for supply chain security. The project provides a policy library, quick start guides, and a playground for testing policies. Key features include:  
- Kubernetes admission control and background scanning  
- Integration with existing DevOps tools  
- Open-source license and active community contributions  
- Supply chain security for container images  

## Key Points
- Enables policy enforcement for Kubernetes resources through validation, mutation, and cleanup.  
- Supports supply chain security via container image signature verification.  
- Integrates with existing Kubernetes workflows and tools like `kubectl` and Git.  
- Maintains a growing community with over 7,600 stars and active development.

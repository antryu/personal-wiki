# Dapr - Distributed Application Runtime

> Dapr is a portable runtime for building distributed applications across cloud and edge, combining event-driven architecture with workflow orchestration. It offers a set of APIs that simplify the development of microservices by providing built-in best practices and patterns, enhancing developer productivity by 20-40%.  
> Last updated: 2026-05-15

## Overview
Dapr is a CNCF-graduated project that provides a set of integrated APIs designed to simplify the development of distributed applications. It abstracts away the complexity of distributed systems by offering out-of-the-box features such as pub/sub, state management, secret stores, external configuration, bindings, actors, distributed locking, and cryptography. Dapr supports event-driven architecture and workflow orchestration, making it easier for developers to build secure and reliable applications.

The project is written in Go and licensed under the Apache License 2.0. It has gained popularity within the cloud-native community, with over 25,700 stars and 2,073 forks on GitHub. Dapr is designed to be platform-agnostic and can be deployed in various environments, including Kubernetes, containers, and serverless platforms.

## Key Points
- Dapr abstracts common distributed system patterns into APIs, reducing boilerplate code and increasing developer productivity.
- It supports a wide range of features including state management, pub/sub messaging, secret management, and workflow orchestration.
- Dapr is a CNCF-graduated project, indicating its maturity and community adoption.

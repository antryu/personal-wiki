**Core Philosophy**
Compound Engineering inverts traditional development by:
1. Prioritizing 80% planning/review over 20% execution
2. Codifying knowledge to reduce future work complexity
3. Using multi-agent systems for quality control
4. Creating self-improving workflows through documentation

**Workflow Cycle**
```
Brainstorm -> Plan -> Work -> Review -> Compound -> Repeat
```

**Key Commands**
| Command | Purpose |
|---------|---------|
| `/ce:ideate` | Discover high-impact improvements via adversarial filtering |
| `/ce:brainstorm` | Interactive Q&A to refine ideas into requirements |
| `/ce:plan` | Generate technical implementation plans |
| `/ce:work` | Execute plans with worktree tracking |
| `/ce:review` | Multi-agent code review before merging |
| `/ce:compound` | Document learnings for future reuse |

**Installation**
- **Claude Code**:  
  ```bash
  /plugin marketplace add EveryInc/compound-engineering-plugin
  /plugin install compound-engineering
  ```
- **Cursor**:  
  ```text
  /add-plugin compound-engineering
  ```
- **Other Platforms**:  
  Converts to OpenCode, Codex, Droid, Pi, Gemini, Copilot, and 5+ formats via CLI:
  ```bash
  bunx @every-env/compound-plugin install compound-engineering --to [format]
  ```

## Key Points
- Inverts traditional engineering debt accumulation by prioritizing planning over execution

- Supports 10+ AI coding platforms through format conversion

- Compounds knowledge through documentation between workflow cycles

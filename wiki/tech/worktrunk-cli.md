# Worktrunk CLI

> Worktrunk is a CLI for Git worktree management, designed for parallel AI agent workflows. It simplifies managing multiple worktrees, making it as intuitive as working with Git branches. Since its release in early 2026, it has become the most popular Git worktree manager due to its clean design and robust features.
> Last updated: 2026-04-17

## Overview

Worktrunk is a command-line interface (CLI) tool written in Rust, aimed at managing Git worktrees efficiently. It is specifically tailored for developers and teams using AI agents to run multiple parallel workflows without interference.

The tool's core functionality revolves around three primary commands that streamline the creation, management, and cleanup of Git worktrees. Worktrunk enhances the developer experience by automating common workflows through hooks and simplifying interactions with multiple branches and changesets.

Git worktrees are a native Git feature that allows a repository to have multiple working directories. This is especially useful when working with AI agents such as Claude Code and Codex, which can operate autonomously on different tasks. Worktrunk makes it easier to manage these parallel processes by abstracting the complexity of worktree management into simple, intuitive commands.

Worktrunk is hosted on GitHub, has a vibrant open-source community, and is maintained under a dual MIT/Apache 2.0 license. It has gained significant traction in the developer ecosystem and is backed by continuous integration and automated testing.

## Key Points

- Worktrunk simplifies Git worktree management, making it as easy as working with Git branches.
- It supports parallel AI agent workflows, allowing developers to manage multiple autonomous tasks simultaneously.
- The tool includes quality-of-life features like hooks to automate local workflows and improve productivity.
- Built in Rust, it is performant and reliable, with a strong focus on user experience and maintainability.
- Worktrunk has been widely adopted and is actively maintained, with a growing community and documentation available at [worktrunk.dev](https://worktrunk.dev).

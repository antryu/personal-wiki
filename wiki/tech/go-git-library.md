# Go Git Library

> A pure Go implementation of Git offering low-level and high-level APIs for repository manipulation. Notably used by Gitea, Keybase, and Pulumi.  
> Last updated: 2026-04-18

## Overview
- **Project**: [go-git/go-git](https://github.com/go-git/go-git)  
- **Stars**: 7,363 | **Forks**: 918  
- **Language**: Go | **License**: Apache 2.0  
- **Created**: 2015 | **Last Update**: 2026-04-18  
- **Topics**: `git`, `go-git`, `git-library`  
- **Open Issues**: 189  

**Key Features**:  
- Pure Go implementation with no external dependencies  
- Supports both "plumbing" (low-level) and "porcelain" (high-level) Git operations  
- Customizable storage via `Storer` interface  
- Full compatibility with standard Git workflows  
- Actively maintained by original authors and [gitsight](https://github.com/gitsight)  

**Project Status**:  
- Resolved legal issues from 2019-2020  
- Backed by a new corporate sponsor for ongoing development  
- Used at scale in production systems  

**Example Use Cases**:  
- Cloning repositories to disk or memory  
- Managing branches and commit histories  
- Implementing custom Git storage solutions  

## Key Points
- Pure Go implementation avoids external binaries  
- Supports in-memory repositories for testing  
- Compatibility with standard Git commands  
- Actively maintained with corporate sponsorship  
- Used by major projects like Gitea and Keybase

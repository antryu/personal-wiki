# Telegraf - Metrics and Logs Agent

> Telegraf is an agent for collecting, processing, aggregating, and writing metrics, logs, and other arbitrary data. It is notable for its extensive plugin ecosystem and ease of deployment, making it a versatile tool for telemetry data collection and transmission.  
> Last updated: 2026-05-08

## Overview

Telegraf is an open-source data collection agent developed by InfluxData, written in Go and licensed under the MIT License. It is designed to collect, process, and transmit metrics and logs from various sources to a wide range of destinations. Telegraf's modular architecture allows users to select and configure plugins to suit their monitoring and telemetry requirements.

Key features include a rich plugin ecosystem with over 300 plugins, support for TOML-based configuration, and the ability to compile into a standalone binary. Telegraf supports system monitoring, log tailing, message queuing, cloud services, and more. It is widely used in both small and large-scale environments for time-series data collection and log management.

## Key Points
- Telegraf offers over 300 plugins for collecting, processing, and sending telemetry data.
- It supports a wide range of sources and destinations, including system metrics, logs, message brokers (Kafka, MQTT), and cloud services.
- Written in Go, it compiles into a static binary with no external dependencies, facilitating easy deployment.
- Configured via TOML for clarity and usability.
- Maintained by a large and active open-source community with over 1,200 contributors.

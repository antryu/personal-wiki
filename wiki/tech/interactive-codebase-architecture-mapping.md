# Interactive Codebase Architecture Mapping

> CodeBoarding provides interactive architecture diagrams for codebases by combining static analysis with LLM reasoning. It helps developers and AI agents understand codebase structures, generate documentation, and visualize changes.
> Last updated: 2026-05-15

## Overview
CodeBoarding is a tool that enables developers and AI agents to visualize and understand complex codebases through interactive architecture diagrams, component-level documentation, and navigable outputs. It uses a combination of static code analysis and large language model (LLM) reasoning to create visual and textual representations of a codebase's architecture. These outputs can be used in IDEs, CI pipelines, and documentation systems, making it easier to understand, maintain, and extend software projects. CodeBoarding also supports incremental updates, so it only re-analyzes parts of the codebase that have changed, reducing processing time.

CodeBoarding supports multiple programming languages including TypeScript, Python, Java, Go, PHP, Rust, and C#. The project is open-source and available as a VS Code extension and GitHub Action.

## Key Points
- Combines static analysis with LLM reasoning to generate architecture diagrams.
- Supports multiple languages and IDE integrations like VS Code and Open VSX.
- Generates documentation, Mermaid diagrams, and incremental updates.

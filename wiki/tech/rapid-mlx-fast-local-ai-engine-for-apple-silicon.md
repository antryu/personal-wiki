# Rapid-MLX: Fast Local AI Engine for Apple Silicon

> Rapid-MLX is the fastest local AI engine for Apple Silicon, offering performance up to 4.2x faster than Ollama. It supports 0.08s cached TTFT, 100% tool calling, and acts as a drop-in OpenAI API replacement. It works seamlessly with tools like Claude Code, Cursor, and Aider.
> Last updated: 2026-05-04

## Overview
Rapid-MLX is a high-performance local AI engine designed specifically for Apple Silicon devices. It enables users to run large language models (LLMs) directly on their Macs, avoiding cloud dependencies and API costs. The engine is optimized for speed and efficiency, supporting a wide range of models such as Qwen, Nemotron-Nano, and DeepSeek, while delivering high performance even with large models like 122B and 158B parameters.

It integrates tool calling, prompt caching, and reasoning separation, making it ideal for developers and AI practitioners who need local, fast, and reliable AI capabilities. Rapid-MLX is API-compatible with OpenAI, allowing developers to use it as a drop-in replacement in their existing workflows.

## Key Points
- **Optimized for Apple Silicon:** Rapid-MLX is tailored for M1, M2, M3, and M4 chips, delivering maximum performance on macOS devices.
- **High Speed and Low Latency:** Achieves up to 160 tokens per second (tok/s) and 0.08s cached TTFT, making it one of the fastest local AI engines.
- **Supports a Range of Models:** Works with models like Gemma 4 26B, Nemotron-Nano 30B, Qwen3.6 35B, DeepSeek V4 Flash 158B, and more.
- **Tool Calling and Prompt Caching:** Enables efficient AI workflows with built-in support for tool calling and reasoning separation.
- **OpenAI API Compatible:** Rapid-MLX supports the OpenAI API specification, making it compatible with tools like Cursor, Claude Code, and Aider.
- **Easy Setup:** Comes with Homebrew and pip installers, as well as an automated one-liner script for quick setup.

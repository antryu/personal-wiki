# Ruby Autoformatter in Rust

> rubyfmt is a fast, opinionated Ruby code formatter written in Rust. It aims to provide a consistent and clean code style by automatically formatting Ruby code. Notable for its performance and adoption in the Ruby ecosystem, rubyfmt is widely used by developers and supported in many popular code editors.
> Last updated: 2026-05-09

## Overview

**rubyfmt** is an autoformatter for Ruby code written in Rust. It provides a fast and efficient way to ensure code consistency across Ruby projects. The tool is notable for its performance, reliability, and integration with popular development tools such as editors and version control systems. rubyfmt is designed to be opinionated, enforcing a specific Ruby code style that aligns with common Ruby conventions and best practices.

The project has a strong community and is actively maintained, as evidenced by the 1,183 stars and 57 forks on GitHub. It supports multiple installation methods, including via Homebrew and from source, and is compatible with various editors including Visual Studio Code, Neovim, and JetBrains IDEs.

rubyfmt also includes features like header comments for per-file formatting control, a `.rubyfmtignore` file to exclude specific files or directories, and command-line flexibility for batch formatting.

## Key Points

- Written in **Rust**, offering high performance and memory safety.
- Opinionated formatting rules that align with common Ruby practices.
- Supports multiple **installation methods**, including Homebrew and source builds.
- Integrates with popular **code editors**, including Visual Studio Code, Neovim, and JetBrains IDEs.
- Provides **header comments** for fine-grained control over file formatting.
- Respects `.rubyfmtignore` and `.gitignore` files to skip specific files.
- Available to try online via [rubyfmt.run](https://rubyfmt.run).

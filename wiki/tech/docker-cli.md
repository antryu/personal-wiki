# Docker CLI

> The Docker CLI repository contains the command-line interface for Docker, enabling users to manage containers and images efficiently. It's notable for being the official CLI for Docker, maintained by the Docker community and actively developed.  
> Last updated: 2026-05-16

## Overview

The `docker/cli` repository is the home of the Docker CLI, a crucial tool for developers and system administrators who work with Docker. This CLI allows users to interact with Docker engines, build images, manage containers, and deploy applications.

The repository is written in Go and licensed under the Apache License 2.0. It has been actively maintained since its creation on April 12, 2017, and last updated on May 16, 2026. The project has gained significant popularity, with over 5,800 stars and 2,100 forks on GitHub.

The CLI can be built from source using Docker itself. Developers can use multiple build targets to compile binaries for different platforms, and the project includes linting, testing, and unit test capabilities to ensure quality and stability.

The repository also includes a development environment that can be launched in a container, making it easy to work with the CLI codebase without needing local dependencies.

## Key Points
- The Docker CLI is the official interface for managing Docker engines, images, and containers.
- It is developed in Go and uses Docker for its build and development processes.
- The project supports cross-platform builds and includes a robust test suite.

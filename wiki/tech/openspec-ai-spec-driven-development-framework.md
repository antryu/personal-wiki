# OpenSpec: AI Spec-Driven Development Framework

> OpenSpec is a spec-driven development (SDD) framework for AI coding assistants, enabling scalable, iterative workflows for teams and individuals. It emphasizes fluid, artifact-guided processes to streamline feature planning, implementation, and documentation.  
> Last updated: 2026-04-15

## Overview  
OpenSpec is a TypeScript-based framework developed by Fission-AI, designed to structure AI-assisted development around specifications. With 40,273 stars and 2,747 forks, it supports 25+ tools and scales from personal projects to enterprises. Key features include:  

- **Artifact-Guided Workflow**: Users initiate changes via commands like `/opsx:propose`, which generates structured artifacts (e.g., proposals, specs, design docs, task checklists).  
- **Iterative Philosophy**: Prioritizes flexibility over rigidity, with workflows like `/opsx:apply` for implementation and `/opsx:archive` for versioning.  
- **Dashboard Integration**: Visualizes project changes and progress via a built-in dashboard.  
- **Team Collaboration**: Offers dedicated Slack access for enterprise teams and active community support via Discord.  

The framework requires Node.js 20.19.0+ and supports package managers like pnpm, yarn, and bun. A quick start guide initializes the tool in any project directory, while expanded workflows cater to advanced use cases.  

## Key Points  
- **Fluid, Iterative Workflow**: Combines spec-driven planning with AI automation for agile development.  
- **Scalability**: Designed for personal projects and enterprise teams, with tools for onboarding, bulk archiving, and synchronization.  
- **MIT License**: Open-source with active development (281 open issues, last update: 2026-04-15).

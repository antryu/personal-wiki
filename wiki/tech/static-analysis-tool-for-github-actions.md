# Static Analysis Tool for GitHub Actions

> This repository provides a powerful static analysis tool for GitHub Actions, helping users identify and fix common security issues in CI/CD workflows. With a large number of stars and ongoing development, it has become a notable open-source project in the security and DevOps space.  
> Last updated: 2026-05-13

## Overview

The `zizmor` project is a static analysis tool for GitHub Actions, designed to detect a wide range of common security issues in CI/CD pipelines. Written in Rust and distributed under the MIT License, it aims to improve the security and reliability of workflows by catching vulnerabilities such as template injection, credential leaks, excessive permissions, and impostor commits.

The project was initiated on 2024-08-19 and has since attracted a large user base, with over 4,799 stars and 185 forks. It is actively maintained, with the latest update on 2026-05-13. The repository is tagged with relevant topics such as `github-actions`, `security`, `security-tools`, and `static-analysis`.

Zizmor supports installation via multiple package managers and platforms, and it features a vibrant community with a Discord server, a comprehensive documentation site, and a clear contributing guide. It is backed by sponsors like Grafana Labs, Trail of Bits, Shipfox, Kusari, and Tracebit.

## Key Points

- Detects critical security vulnerabilities in GitHub Actions workflows, including template injection and credential leakage.
- Written in Rust for performance and safety.
- Open source under the MIT License, with a strong and growing community.
- Actively maintained, with a high number of stars and forks.
- Provides detailed documentation and usage guides for ease of adoption.

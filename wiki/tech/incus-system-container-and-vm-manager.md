# Incus - System Container and VM Manager

> Incus is a powerful system container and virtual machine manager that offers a unified experience for running and managing full Linux systems. It is notable for its scalability, from a single machine to data center clusters, and its support for a wide range of Linux distributions.
> Last updated: 2026-05-04

## Overview
Incus is a modern and secure system container and VM manager developed by the Linux Containers community. It provides a REST API-based platform for managing Linux workloads, including containers and virtual machines, in an efficient and scalable manner. Incus is suitable for both development and production environments and can be used to set up a system resembling a small private cloud.

Incus began as a community fork of Canonical's LXD after the project was taken over by Canonical and has since been adopted and maintained by the Linux Containers community. It is an open-source project licensed under Apache 2.0 and is free from CLA (Contributor License Agreement) requirements. The project is actively developed in Go and offers a migration tool for users transitioning from LXD.

Users can manage Incus via a web interface, command-line tool, or REST API. It supports a wide array of Linux distributions and is designed to be both secure and flexible. Incus offers extensive documentation and community support through its official website and GitHub repository.

## Key Points
- Incus provides a unified experience for managing containers and virtual machines with a powerful REST API.
- It supports a wide range of Linux distributions, including official Ubuntu and community-provided images.
- Incus is a community-driven project, open-source, and licensed under Apache 2.0.
- It scales from a single machine setup to full data center clusters.
- Incus includes a migration tool for users transitioning from LXD to Incus.

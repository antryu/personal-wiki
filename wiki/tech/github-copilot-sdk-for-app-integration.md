# GitHub Copilot SDK for App Integration

> GitHub has released a multi-platform SDK for integrating GitHub Copilot Agent into apps and services. This SDK provides a programmable interface for embedding Copilot's agentic workflows, allowing developers to leverage the same agent runtime used in Copilot CLI. The SDK is now publicly available in preview for Python, TypeScript, Go, .NET, and Java.  
> Last updated: 2026-05-08

## Overview
The **GitHub Copilot SDK** is a development tool that enables developers to embed the GitHub Copilot Agent into their applications and services using a multi-platform, programmable interface. It exposes the same engine as the Copilot CLI, allowing developers to leverage agent workflows without the need to build their own orchestration. With the SDK, developers can invoke agent behavior programmatically, and let Copilot handle planning, tool invocation, and file edits.

The SDK is available in preview for multiple programming languages including Python, TypeScript, Go, .NET, and Java. Each SDK includes installation instructions, a cookbook for usage, and a dedicated directory within the repository. This makes it easy for developers to integrate and begin using Copilot's capabilities within their own codebase.

The project is actively maintained, with updates as recent as May 8, 2026. As of now, the repository has 8,621 stars, 1,165 forks, and 223 open issues on GitHub.

## Key Points
- The GitHub Copilot SDK provides an agent runtime that can be programmatically invoked.
- It supports multiple programming languages including Python, TypeScript, Go, .NET, and Java.
- Developers can use the SDK without needing to build their own orchestration systems.
- The SDK includes installation guides and cookbooks for each language.
- It is available under an MIT License and is in public preview.

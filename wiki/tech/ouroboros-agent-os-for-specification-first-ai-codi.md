# Ouroboros: Agent OS for Specification-First AI Coding

> Ouroboros is an Agent OS for AI coding that transforms non-deterministic agent work into a replayable, observable, and policy-bound execution contract. It enables a structured, specification-first workflow to turn vague ideas into verified codebases. Notable for its support across multiple AI platforms and its focus on clarity and verification.
> Last updated: 2026-05-04

## Overview
Ouroboros is a local-first runtime layer designed to streamline AI coding workflows by replacing ad-hoc prompting with a specification-first approach. It provides a Socratic interview process, an immutable seed specification to lock in intent, and a three-stage evaluation gate to ensure quality. The system is compatible with multiple AI coding tools, including Claude Code, Codex CLI, OpenCode, and Hermes. Installation is straightforward, with support for various setup methods and runtime environments.

Key components of Ouroboros include:

- **Structured workflow**: Interview, crystallize, execute, evaluate, evolve.
- **Platform compatibility**: Supports multiple AI coding platforms with automatic detection.
- **Verification**: Enforces clarity, and uses automated gates to ensure output quality.
- **Install options**: One-command installation, plugin-based for Claude Code, and pip-based for full control.

## Key Points
- Replaces vague prompts with a Socratic interview to uncover hidden assumptions.
- Ensures consistent results with an immutable specification before code generation.
- Offers automated three-stage evaluation for verification, reducing manual QA.
- Provides multiple installation methods and runtime compatibility.
- Compatible with major AI coding tools and supports future extensibility.

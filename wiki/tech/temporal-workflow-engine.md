# Temporal Workflow Engine

> Temporal is a durable execution platform for building scalable, reliable applications by managing workflows and activities with automatic failure handling. Notable for its origins as a fork of Uber's Cadence and active development by Temporal Technologies.  
> Last updated: 2026-04-18

## Overview
Temporal enables developers to design resilient applications through durable workflow execution. It automatically retries failed operations and handles intermittent failures, ensuring reliability in distributed systems. Originally forked from Uber's Cadence, it is now maintained by Temporal Technologies, a startup founded by Cadence creators.  

The project is written in Go, licensed under MIT, and hosts 19,691 stars and 1,488 forks on GitHub. It supports workflows in multiple languages (e.g., Go, Java) and includes tools like a CLI, Web UI for monitoring workflows, and community resources (forum, Slack). Key features include cron job scheduling, microservice orchestration, and advanced workflow management.  

Repository details:

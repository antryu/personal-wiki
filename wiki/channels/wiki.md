# #wiki 위키

## 핵심 주제 요약
- 사용자가 Obsidian Vault에 저장된 데이터를 접근하려고 시도했으나, 워크스페이스 범위를 벗어난 경로로 인해 접근이 불가능한 상황.
- LLM (Large Language Model)에 대한 질문과 함께, 현재 사용 중인 모델 (`qwen3`) 및 다른 모델 변경 방법에 대한 안내.
- M4 로컬 저장소의 데이터 접근을 위한 대안 제시.

## 주요 의사결정 사항
- 워크스페이스 범위를 벗어난 파일 접근은 불가능하므로, 파일 이동 또는 다른 방법으로 접근해야 함.
- `qwen3` 모델 사용 중이나, 필요에 따라 `sonnet` 등 다른 모델로 변경 가능.

## 공유된 링크/리소스
- 없음

## 미완료 작업
1. Obsidian Vault의 데이터를 워크스페이스 범위 내로 이동.
2. M4 로컬 저장소의 데이터 접근 방법 확인 및 실행.

## 인사이트
- **워크스페이스 범위**: 워크스페이스 외부에 있는 파일은 보안상의 이유로 접근이 제한됩니다. 이를 해결하기 위해 파일을 워크스페이스 내로 이동하거나, 명령어를 사용하여 데이터를 읽어야 합니다.
- **LLM 모델 변경**: 다양한 LLM 모델이 있으며, 각 모델은 특정 기능에 최적화되어 있습니다. 예를 들어, `qwen3`는 한국어 처리와 문서 검색에 강한 반면, `sonnet`은 자연어 생성에 특화되어 있습니다.
- **데이터 접근 방법**: 로컬 저장소의 데이터를 워크스페이스로 이동하거나, 명령어를 사용하여 파일을 읽거나 서버를 실행할 수 있습니다. 이러한 방법들은 보안과 효율성을 동시에 유지하는 데 도움이 됩니다.

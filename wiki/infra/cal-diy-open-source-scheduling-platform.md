# Cal.diy Open Source Scheduling Platform

> A community-driven, MIT-licensed open-source scheduling platform forked from Cal.com, designed for self-hosting with no commercial dependencies. Notable for its full transparency and removal of enterprise features to prioritize accessibility and control for individual users.  
> Last updated: 2026-04-16

## Overview
**Cal.diy** is a self-hosted scheduling infrastructure project, fully open source under the MIT License. It originated as a fork of Cal.com in 2021, stripping out all proprietary "Enterprise Edition" code to provide a community-maintained alternative. The platform emphasizes full code transparency, requiring no license keys or Cal.com account for operation.  

Key technical details include:
- **Repository**: [GitHub](https://github.com/calcom/cal.diy) with 41,322 stars and 12,791 forks
- **Language**: TypeScript with a Next.js/Tailwind CSS frontend and PostgreSQL/Prisma backend
- **Ecosystem**: Built on the T3 Stack, leveraging tRPC, Daily.co for video, and Turborepo for monorepo management
- **Maintenance**: Actively updated (last commit: 2026-04-16) with 1,303 open issues and community contribution guidelines

The project explicitly warns users that it requires advanced technical knowledge for deployment and is recommended for non-production/personal use only. Commercial users are directed to Cal.com's enterprise offerings.

## Key Points
- **No enterprise features**: Teams, SSO/SAML, workflows, and analytics removed
- **No license restrictions**: Fully MIT-licensed without "Open Core" limitations
- **Self-hosted only**: No managed/cloud hosting available; requires PostgreSQL, Node.js 18+, and server administration expertise
- **Community governance**: Contributions merge directly into the main repository
- **Commercial alternative**: Cal.com (hosted) or on-prem enterprise access via [cal.com/sales](https://cal.com/sales)

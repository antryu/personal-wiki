# M4Max128G 서버 — S&P500 자동매매 시스템 개발

> M4Max128G 봇 채널에서 진행된 S&P500 MES(Micro E-mini) 자동매매 시스템의 실전 운용, 백테스트 분석, 전략 파라미터 최적화 과정을 기록한 문서. 키움증권 CME 선물 기반 VBA 로직과 Python 백테스트 환경 간의 정합성 확보 및 수익 구조 개선이 핵심 주제.
> Last updated: 2026-04-05

## Overview

2026년 2월부터 실전 투입된 S&P500 MES 자동매매 시스템의 운용 결과를 분석하고, 전략 파라미터를 최적화하는 과정이 진행되었다. 초기 2주간 8거래에서 +$21.25(수수료 전)를 기록했으나, 신호사용료($200~250/월)와 거래비용을 감안하면 실질 적자 구조임이 드러났다. 이후 트레일링 스탑 파라미터 조정, ATR 기반 동적 손절 도입, 레짐 로직의 Python 백테스트 반영 등 체계적인 개선 작업이 수행되었다.

## Key Topics

### 실전 거래 결과 분석 (2/18~2/27)

| 2/19 | 숏 | -6.00 | HIGH |
| 2/19 | 롱 | +7.00 | - |
| 2/20 | 롱 | -3.75 | LOW |

- **2주 실질 손익**: **-$108.75**
- 승률 37.5% (3/8), 손절 일관적(-3.75pt), 2/25 하루가 전체 수익의 대부분

### 비용 구조 분석

- MES 1계약 = $5/pt
- 왕복 수수료: $2 (0.4pt)
- 슬리피지: 0.5pt (왕복)
- **거래당 총 비용: 3pt = $3.75**
- 신호사용료: **$200~250/월**
- 평균 수익 +0.53pt/거래 → 거래당 순이익 $0.66 → 월 $200 고정비 커버 불가능
- **손익분기: 303거래/월** (현실적으로 불가)

### ATR 기반 동적 손절 도입

기존 고정 손절(15/25틱 = 3.75/6.25pt)이 시장 노이즈에 빈번히 걸리는 문제 확인.

**고정 SL vs ATR SL 비교**:

| 시나리오 | 승률 | PF | 월 순수익(신호비 후) |
|----------|------|------|-----------------|
| 고정SL (Long only) | - | 1.91 | -$144 |
| 고정SL (양방향) | - | 1.99 | -$91 |
| **ATR SL 3×20** | **96.8%** | **15.97** | **+$217** |

**VBA 수정 포인트**:
- `SetRegimeParameters`에서 `DF25 = RegimeStopLoss` → `DF25 = DC25 * 12` (ATR_20_AVG × 3, 틱 단위)
- 최소 10틱(2.5pt), 최대 60틱(15pt) 제한
- `CalcAdaptiveTrailingStop` Else 분기: `RegimeStopLoss / 4` → `actualSL / 4` (DF25 셀 참조)

### VBA-Python 정합성 문제

레짐 로직(MA20/MA200 + ATR 변동성 기반 4단계)이 VBA(키움 엑셀)에만 존재하고 Python 백테스트에는 미구현이었음.

| 기능 | Python | VBA(키움) |
|------|--------|---------|
| BB(40,2) 신호 | ✅ | ✅ |
| 레짐 판단 | ❌→✅구현 | ✅ |
| 레짐별 SL/TP | ❌→✅구현 | ✅ |
| 트레일링 스탑 | ✅ (기본 OFF) | ✅ |
| ATR 손절 | ✅ | ❌→✅수정 |

Python에 `src/regime_strategy.py` + `run_regime_backtest.py` 구현하여 VBA와 동일 조건 백테스트 가능하도록 정합성 확보.

### 레짐 시스템 상세

| 레짐 | 조건 | 상단 이탈 | 하단 이탈 |
|------|------|---------|---------|
| UP_LOW | MA20>MA200, 저변동 | 롱 ✅ | 롱 ✅ |
| UP_HIGH | MA20>MA200, 고변동 | 롱 ✅ | 롱 ✅ |
| DOWN_LOW | MA20<MA200, 저변동 | 숏 ✅ | 숏 ✅ |
| DOWN_HIGH | MA20<MA200, 고변동 | 숏 ✅ | 롱 ✅ |

시장 상태에 따라 자동으로 롱/숏 전환되는 구조. Long only가 아님.

### 청산 체계 (우선순위)

1. **시간 청산**: 휴장 전 / 경제지표 발표(실업률, CPI, PPI) 시 강제 청산
2. **손절**: ZZ <= -DF25 (ATR 기반 동적 SL)
3. **트레일링 스탑**: 수익 >= 15틱 → 5틱 간격 추적 (ZZ_Max 갱신 시)
4. **BB 청산**: 종가가 BB 반대편/10MA 복귀 시 (진입 후 충분한 시간 경과)

### TP(익절) 미사용 이슈

`DG25`에 RegimeTakeProfit(20~30틱) 저장되지만 실제 청산 로직에서 참조하지 않음. 익절은 100% 트레일링 스탑에 의존.

**Option B 제안**: TP 도달 시 TrailingStep을 3틱으로 타이트하게 조이는 3단계 구조
- 수익 < 15틱 → ATR SL (초기 보호)
- 수익 >= 15틱 → 5틱 트레일링
- 수익 >= TP틱 → 3틱 트레일링 (수익 잠금)

→ 코드 준비 완료, **한 변수씩 변경 원칙에 따라 적용 보류**
